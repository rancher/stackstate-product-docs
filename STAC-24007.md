We want to add to the documentation that dasbhoards are supported on the stackstate-cli tool. What we don’t want is to spell out the details of every sub-command (which would be also a pain to maintain). 

The --help option on the CLI should be detailed enough for users to use any of the commands. For example, the dashboard CRUD commands shouldn’t need any additional documentation. 

Commands that are used in a troubleshooting flow or for a specific purpose should be included (with examples) in the detailed documentation for that user flow (troubleshooting topology or health synchronization, monitor creation, etc.). So the CLI docs page only needs to have a list with the top-level commands and a short description for each so users have an overview of the capabilities of the Cli.

Acceptance criteria

1. Add an overview of CLI top-level commands with a short description, similar to the one in the Rancher docs. There is a page already for the cli @./docs/latest/modules/en/pages/setup/cli/cli-sts.adoc the changes can be done there. 
2. Include the dashboard command behind its feature flag
3. Include the otel mapping commands behind the stackpacks2 feature flag (The feauture flag is environment variable STS_EXPERIMENTAL_STACKPACK set to any value, lets say "true")
