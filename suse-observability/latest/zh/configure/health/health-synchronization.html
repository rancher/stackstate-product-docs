еҒҘеә·еҗҢжӯҘ

жң¬иҠӮжҸҸиҝ°дәҶе°ҶжқҘиҮӘдёҚеҗҢзӣ‘жҺ§зі»з»ҹзҡ„иҮӘе®ҡд№үеҒҘеә·ж•°жҚ®еҗҢжӯҘеҲ° SUSE Observability зҡ„й«ҳзә§дё»йўҳгҖӮ иҝҷдёӘдё»йўҳдё»иҰҒй’ҲеҜ№еёҢжңӣдёҺзҺ°жңүзӣ‘жҺ§зі»з»ҹиҝӣиЎҢиҮӘе®ҡд№үйӣҶжҲҗзҡ„е·ҘзЁӢеёҲгҖӮ еҜ№дәҺзҺ°жҲҗзҡ„зӣ‘жҺ§еҷЁпјҢжӮЁеҸҜд»ҘжҹҘзңӢиҝҷйҮҢгҖӮ

жҰӮиҝ°

еҒҘеә·еҗҢжӯҘе°ҶжқҘиҮӘеӨ–йғЁзӣ‘жҺ§зі»з»ҹзҡ„зҺ°жңүеҒҘеә·жЈҖжҹҘж·»еҠ еҲ° SUSE Observability жӢ“жү‘е…ғзҙ дёӯгҖӮеҒҘеә·ж•°жҚ®еңЁеӨ–йғЁзӣ‘жҺ§зі»з»ҹдёӯдҪҝз”Ёе…¶иҮӘиә«зҡ„ж•°жҚ®е’Ңи§„еҲҷиҝӣиЎҢи®Ўз®—пјҢз„¶еҗҺиҮӘеҠЁеҗҢжӯҘе№¶йҷ„еҠ еҲ° SUSE Observability дёӯзҡ„зӣёе…іжӢ“жү‘е…ғзҙ гҖӮ

и®ҫзҪ®еҒҘеә·еҗҢжӯҘ

SUSE Observability жҺҘж”¶еҷЁ API е°ҶиҮӘеҠЁжҺҘж”¶е’ҢеӨ„зҗҶжүҖжңүдј е…Ҙзҡ„еҒҘеә·ж•°жҚ®гҖӮSUSE Observability дёҚйңҖиҰҒйўқеӨ–й…ҚзҪ®жқҘеҗҜз”ЁеҒҘеә·еҗҢжӯҘпјҢз„¶иҖҢпјҢжҺҘж”¶еҲ°зҡ„еҒҘеә·ж•°жҚ®еә”з¬ҰеҗҲйў„жңҹзҡ„ JSON ж јејҸгҖӮ

жңүе…іеҰӮдҪ•ж‘„еҸ–еҒҘеә·ж•°жҚ®зҡ„иҜҰз»ҶдҝЎжҒҜеҸҜд»ҘеңЁд»ҘдёӢйЎөйқўжүҫеҲ°пјҡ

  • йҖҡиҝҮ SUSE Observability жҺҘж”¶еҷЁ API ж‘„еҸ–еҒҘеә·ж•°жҚ®

еҒҘеә·еҗҢжӯҘз®ЎйҒ“

еҒҘеә·еҗҢжӯҘжЎҶжһ¶зҡ„е·ҘдҪңеҺҹзҗҶеҰӮдёӢпјҡ

  • еҒҘеә·ж•°жҚ®иў«еҸ‘йҖҒеҲ° SUSE ObservabilityпјҢе№¶йҖҡиҝҮжҺҘж”¶еҷЁ API иҝӣиЎҢж‘„еҸ–гҖӮ

  • дёҺж‘„еҸ–зҡ„еҒҘеә·жЈҖжҹҘзӣёе…ізҡ„ SUSE Observability жӢ“жү‘е…ғзҙ ж №жҚ®д»ҘдёӢеҶ…е®№иҝӣиЎҢиҜҶеҲ«е’Ңз»‘е®ҡпјҡ

    • еңЁжӢ“жү‘еҗҢжӯҘжңҹй—ҙиҺ·еҫ—зҡ„жӢ“жү‘ж ҮиҜҶз¬ҰгҖӮ

    • жқҘиҮӘж‘„еҸ–зҡ„еҒҘеә·иҙҹиҪҪзҡ„`topologyElementIdentifier`гҖӮ

  • SUSE Observability и·ҹиёӘжӢ“жү‘е…ғзҙ е’ҢеҒҘеә·жЈҖжҹҘзҡ„еҸҳеҢ–пјҢд»ҘдҝқжҢҒжңҖж–°дҝЎжҒҜгҖӮ

дёҖиҮҙжҖ§жЁЎеһӢ

SUSE Observability еҒҘеә·еҗҢжӯҘдҫқиө–дәҺдёҚеҗҢзҡ„дёҖиҮҙжҖ§жЁЎеһӢпјҢд»ҘзЎ®дҝқжқҘиҮӘеӨ–йғЁзӣ‘жҺ§зі»з»ҹзҡ„ж•°жҚ®дёҺ SUSE Observability жүҖжҺҘж”¶е’ҢжҳҫзӨәзҡ„ж•°жҚ®зӣёеҢ№й…ҚгҖӮдёҖиҮҙжҖ§жЁЎеһӢеңЁ "health" еұһжҖ§дёӯжҢҮе®ҡпјҢжҲ–иҖ…еңЁе°ҶеҒҘеә·ж•°жҚ®еҸ‘йҖҒеҲ° SUSE Observability ж—¶дҪңдёә SUSE Observability CLI зҡ„дёҖдёӘеҸӮж•°гҖӮж”ҜжҢҒзҡ„жЁЎеһӢжңүпјҡREPEAT_SNAPSHOTS е’Ң TRANSACTIONAL_INCREMENTSгҖӮ

  • йҮҚеӨҚеҝ«з…§жЁЎеһӢ

  • дәӢеҠЎеўһйҮҸжЁЎеһӢ

REPEAT_SNAPSHOTS дёҖиҮҙжҖ§жЁЎеһӢйҖӮз”ЁдәҺеӨ–йғЁзӣ‘жҺ§зі»з»ҹдёӯжүҖжңүжЈҖжҹҘзҡ„е®ҡжңҹе®Ңж•ҙеҝ«з…§гҖӮSUSE Observability и·ҹиёӘжҜҸдёӘжҺҘж”¶еҝ«з…§дёӯзҡ„жЈҖжҹҘпјҢе№¶еҶіе®ҡжҳҜеҗҰйңҖиҰҒеңЁ SUSE Observability дёӯеҲӣе»әгҖҒжӣҙж–°жҲ–еҲ йҷӨзӣёе…ізҡ„еӨ–йғЁжЈҖжҹҘзҠ¶жҖҒгҖӮдҫӢеҰӮпјҢеҰӮжһңжҹҗдёӘжЈҖжҹҘзҠ¶жҖҒеңЁеҝ«з…§дёӯдёҚеҶҚеӯҳеңЁгҖӮиҜҘжЁЎеһӢжҸҗдҫӣдәҶеҜ№е°ҶеҲ йҷӨе“ӘдәӣеӨ–йғЁжЈҖжҹҘзҡ„е®Ңе…ЁжҺ§еҲ¶пјҢеӣ дёәжүҖжңүеҶізӯ–йғҪжҳҜд»ҺжҺҘж”¶зҡ„еҝ«з…§дёӯжҺЁж–ӯеҮәжқҘзҡ„гҖӮеҜ№дәҺе°ҶеҮәзҺ°еңЁ SUSE Observability дёӯзҡ„еӨ–йғЁжЈҖжҹҘпјҢжІЎжңүд»»дҪ•жӯ§д№үгҖӮ

*еңЁд»ҘдёӢжғ…еҶөдёӢдҪҝз”ЁжӯӨжЁЎеһӢпјҡ*еӨ–йғЁзӣ‘жҺ§зі»з»ҹиғҪеӨҹеңЁзү№е®ҡж—¶й—ҙзӘ—еҸЈеҶ…и®°еҪ•е“Әдәӣе…ғзҙ еӯҳеңЁзҡ„зҠ¶жҖҒпјҢеӣ жӯӨеҸҜд»Ҙдј йҖ’е®Ңж•ҙеҝ«з…§зҡ„еҶ…е®№гҖӮ

*JSON иҙҹиҪҪпјҡ*йҮҚеӨҚеҝ«з…§еҒҘеә·иҙҹиҪҪ жҺҘеҸ—зү№е®ҡеұһжҖ§д»ҘжҢҮе®ҡеҝ«з…§дҪ•ж—¶ејҖе§ӢжҲ–еҒңжӯўгҖӮ

TRANSACTIONAL_INCREMENTS дёҖиҮҙжҖ§жЁЎеһӢж—ЁеңЁз”ЁдәҺжөҒејҸзі»з»ҹпјҢе…¶дёӯд»…е°ҶеўһйҮҸжӣҙж”№дј иҫҫз»ҷ SUSE ObservabilityгҖӮз”ұдәҺж•°жҚ®жІЎжңүйҮҚеӨҚпјҢеӣ жӯӨйҖҡиҝҮзЎ®дҝқж•ҙдёӘз®ЎйҒ“дёӯиҮіе°‘дёҖж¬ЎдәӨд»ҳжқҘз»ҙжҠӨж•°жҚ®дёҖиҮҙжҖ§гҖӮдёәдәҶжЈҖжөӢжҳҜеҗҰжңүд»»дҪ•ж•°жҚ®зјәеӨұпјҢSUSE Observability иҰҒжұӮеҗҢж—¶дј иҫҫжЈҖжҹҘзӮ№е’ҢдёҠдёҖдёӘжЈҖжҹҘзӮ№д»ҘеҸҠ check_statesгҖӮиҜҘжЁЎеһӢиҰҒжұӮеңЁж•ҙдёӘз®ЎйҒ“дёӯиҝӣиЎҢдёҘж јжҺ§еҲ¶пјҢд»ҘзЎ®дҝқжІЎжңүж•°жҚ®дёўеӨұгҖӮ

*еңЁд»ҘдёӢжғ…еҶөдёӢдҪҝз”ЁжӯӨжЁЎеһӢпјҡ*еӨ–йғЁзӣ‘жҺ§зі»з»ҹж— жі•и®ҝй—®жүҖжңүеӨ–йғЁжЈҖжҹҘзҠ¶жҖҒпјҢиҖҢд»…еҹәдәҺдәӢд»¶иҝӣиЎҢе·ҘдҪңгҖӮ

*JSON иҙҹиҪҪпјҡ*е…ғж•°жҚ® repeat_interval е’Ң expire_interval еҜ№дәҺ дәӢеҠЎеўһйҮҸеҒҘеә·иҙҹиҪҪ е№¶дёҚзӣёе…іпјҢеӣ дёәж•°жҚ®жІЎжңүйў„е®ҡд№үзҡ„е‘ЁжңҹжҖ§гҖӮ

еҒҘеә·жөҒе’ҢеӯҗжөҒ

еӨ–йғЁзӣ‘жҺ§зі»з»ҹе°ҶеҒҘеә·ж•°жҚ®еҸ‘йҖҒеҲ° SUSE Observability жҺҘж”¶еҷЁзҡ„еҒҘеә·жөҒдёӯгҖӮжҜҸдёӘеҒҘеә·жөҒиҮіе°‘жңүдёҖдёӘеҢ…еҗ«еҒҘеә·жЈҖжҹҘзҡ„еӯҗжөҒгҖӮ

еҒҘеә·жөҒ

еҒҘеә·жөҒе”ҜдёҖж ҮиҜҶеҒҘеә·еҗҢжӯҘпјҢе№¶е®ҡд№үеҒҘеә·жЈҖжҹҘзҠ¶жҖҒеә”дёҖиө·еӨ„зҗҶзҡ„иҫ№з•ҢгҖӮ

еӯҗжөҒ

еӯҗжөҒеҢ…еҗ«з”ұ SUSE Observability еӨ„зҗҶзҡ„еҒҘеә·жЈҖжҹҘж•°жҚ®гҖӮеңЁеӨ„зҗҶжқҘиҮӘеҲҶеёғејҸеӨ–йғЁзӣ‘жҺ§зі»з»ҹзҡ„еҒҘеә·ж•°жҚ®ж—¶пјҢеҸҜд»Ҙй…ҚзҪ®еӨҡдёӘеӯҗжөҒпјҢжҜҸдёӘеӯҗжөҒеҢ…еҗ«жқҘиҮӘеҚ•дёӘдҪҚзҪ®зҡ„еҒҘеә·еҝ«з…§гҖӮжҜҸдёӘеӯҗжөҒдёӯзҡ„ж•°жҚ®жҳҜеҚҠзӢ¬з«Ӣзҡ„пјҢдҪҶеҜ№е®Ңж•ҙеҒҘеә·жөҒзҡ„еҒҘеә·жЈҖжҹҘзҠ¶жҖҒжңүиҙЎзҢ®гҖӮеҰӮжһңеҚ•дёӘдҪҚзҪ®иҙҹиҙЈжҠҘе‘ҠеҒҘеә·жөҒзҡ„еҒҘеә·жЈҖжҹҘзҠ¶жҖҒпјҢеҸҜд»Ҙд»ҺеҒҘеә·иҙҹиҪҪдёӯзңҒз•Ҙ`sub_stream_id`гҖӮSUSE Observability е°ҶеҒҮи®ҫжүҖжңүеӨ–йғЁеҒҘеә·жЈҖжҹҘеұһдәҺдёҖдёӘй»ҳи®Өзҡ„еӯҗжөҒгҖӮ

йҮҚеӨҚй—ҙйҡ”

еҒҘеә·еҗҢжӯҘжҢүеӯҗжөҒеӨ„зҗҶж‘„еҸ–зҡ„еҒҘеә·ж•°жҚ®гҖӮеңЁ еҒҘеә·иҙҹиҪҪ дёӯжҢҮе®ҡзҡ„йҮҚеӨҚй—ҙйҡ”жҳҜеӨ–йғЁзӣ‘жҺ§зі»з»ҹжүҝиҜәдёҚж–ӯеҸ‘йҖҒе®Ңж•ҙеҝ«з…§пјҢд»ҘдҝқжҢҒ SUSE Observability дёҠж•°жҚ®зҡ„жңҖж–°зҠ¶жҖҒгҖӮиҝҷжңүеҠ©дәҺ SUSE Observability еҗ‘з”ЁжҲ·еұ•зӨәеҒҘеә·еҗҢжӯҘзҡ„жңҖж–°зҠ¶жҖҒгҖӮ

иҝҮжңҹй—ҙйҡ”

иҝҮжңҹй—ҙйҡ”еҸҜз”ЁдәҺй…ҚзҪ®еҒҘеә·еҗҢжӯҘдёӯзҡ„еӯҗжөҒпјҢд»ҘеҲ йҷӨдёҚеҶҚз”ұеӨ–йғЁзі»з»ҹеҸ‘йҖҒзҡ„ж•°жҚ®гҖӮиҝҷеңЁеӯҗжөҒжқҘжәҗеҸҜиғҪиў«йҖҖеҪ№гҖҒSUSE Observability еӣ жӯӨдёҚеҶҚжҺҘж”¶еҲ°ж•°жҚ®зҡ„жғ…еҶөдёӢйқһеёёжңүз”ЁгҖӮеҰӮжһңжІЎжңүиҝҮжңҹй—ҙйҡ”пјҢд№ӢеүҚеҗҢжӯҘзҡ„ж•°жҚ®е°Ҷдјҡж°ёд№…дҝқз•ҷгҖӮ

жЈҖжҹҘзҠ¶жҖҒ

еҒҘеә·жЈҖжҹҘзҠ¶жҖҒз”ұеӨ–йғЁзӣ‘жҺ§зі»з»ҹи®Ўз®—пјҢе№¶еҢ…еҗ«е°Ҷе…¶йҷ„еҠ еҲ°жӢ“жү‘е…ғзҙ жүҖйңҖзҡ„жүҖжңүдҝЎжҒҜгҖӮдёәдәҶиғҪеӨҹе°Ҷе…¶е…·дҪ“еҢ–е№¶йҷ„еҠ еҲ°з»„д»¶дёҠпјҢйңҖиҰҒе°ҶеҒҘеә·зҠ¶жҖҒеҪ’еұһз»ҷзү№е®ҡзҡ„зӣ‘жҺ§еҷЁпјҢеңЁиҝҷз§Қжғ…еҶөдёӢжҳҜдёҖдёӘ еӨ–йғЁзӣ‘жҺ§еҷЁгҖӮ
