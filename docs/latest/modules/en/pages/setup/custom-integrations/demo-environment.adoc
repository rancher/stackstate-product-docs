= Demo Environment for the Generic StackPack
:page-languages: [en]
:revdate: 2026-01-08
:page-revdate: {revdate}
:description: Set up OpenTelemetry Demo data for a custom StackPack

This tutorial creates a small environment for trying the generic StackPack template from xref:/setup/custom-integrations/create.adoc[Create a Custom Integration]. The goal is to send OpenTelemetry Demo telemetry to {stackstate-product-name}, then use the scaffolded StackPack to customize how that telemetry is mapped and presented.

== How the pieces fit together

The demo setup has three parts:

* The {stackstate-product-name} platform stores topology, metrics, traces, monitors, and StackPack configuration.
* The {stackstate-product-name} Agent runs in the Kubernetes cluster where the demo application runs. With the telemetry gateway enabled, it accepts OTLP metrics and traces from workloads in the cluster and forwards them to the platform.
* The OpenTelemetry Demo runs as the example application. Its services export telemetry directly to the Agent telemetry gateway service.

The data path is:

[source,text]
----
OpenTelemetry Demo services
  -> {stackstate-product-name} Agent telemetry gateway
  -> {stackstate-product-name} platform
----

== Prerequisites

* A Kubernetes cluster with `kubectl` access
* Helm 3 installed
* A running {stackstate-product-name} instance
* The xref:/setup/cli/cli-sts.adoc[{stackstate-product-name} CLI] configured for that instance

For a small self-hosted setup, follow xref:/installing-suse-obs.adoc[Installing {stackstate-product-name}] and use a non-HA sizing profile such as `trial` or `10-nonha` if that fits your environment.

== Install the Agent with the telemetry gateway

Install the {stackstate-product-name} Agent in the same Kubernetes cluster where you will deploy the OpenTelemetry Demo.

. In the {stackstate-product-name} UI, install the Kubernetes StackPack for the target cluster. For the full flow, see xref:/k8s-quick-start-guide.adoc[Quick start guide].
. Copy the Helm command generated by the Kubernetes StackPack instructions.
. Add a values file that enables the OpenTelemetry pipeline and telemetry gateway.

The OpenTelemetry Demo sends OTLP data to the telemetry gateway in the {stackstate-product-name} Agent. Enable the gateway in the Agent Helm values:

[source,bash]
----
cat > agent-otel-values.yaml <<'EOF'
otel:
  enabled: true
  telemetryGateway:
    enabled: true
EOF
----

Add `--values agent-otel-values.yaml` to the generated Agent Helm command. The final command should look like this, with the cluster-specific values from the UI-generated command preserved:

[source,bash]
----
helm upgrade --install suse-observability-agent suse-observability/suse-observability-agent \
  --namespace suse-observability-agent \
  --create-namespace \
  --values agent-otel-values.yaml \
  <cluster-specific values from the Kubernetes StackPack instructions>
----

If the Agent is already installed, run the same Helm command with `--values agent-otel-values.yaml` added.

The effective Agent values must include:

[source,yaml]
----
otel:
  enabled: true
  telemetryGateway:
    enabled: true
----

For the telemetry gateway details and endpoint names, see xref:/setup/otel/telemetry-gateway.adoc[Telemetry gateway].

Verify that the telemetry gateway is running:

[source,bash]
----
kubectl -n suse-observability-agent get deploy,svc | grep telemetry-gateway
----

This tutorial assumes the default telemetry gateway service name and namespace:

[source,text]
----
http://suse-observability-agent-otel-telemetry-gateway.suse-observability-agent.svc.cluster.local:4317
----

== Deploy the OpenTelemetry Demo

Create a namespace and values file for the public OpenTelemetry Demo Helm chart.

The values file disables the demo chart's bundled collector and local observability backends, then points the demo services at the {stackstate-product-name} Agent telemetry gateway. This keeps the setup focused on the data path used by this tutorial.

[source,bash]
----
cat > otel-demo-values.yaml <<'EOF'
default:
  envOverrides:
    - name: OTEL_COLLECTOR_NAME
      value: suse-observability-agent-otel-telemetry-gateway.suse-observability-agent.svc.cluster.local

opentelemetry-collector:
  enabled: false

prometheus:
  enabled: false

grafana:
  enabled: false

jaeger:
  enabled: false

opensearch:
  enabled: false
EOF
----


Install the demo:

[source,bash]
----
helm repo add open-telemetry https://open-telemetry.github.io/opentelemetry-helm-charts
helm repo update

helm upgrade --install opentelemetry-demo open-telemetry/opentelemetry-demo \
  --namespace telemetry-demo \
  --create-namespace \
  --values otel-demo-values.yaml
----

The `telemetry-demo` namespace matches the scaffolded StackPack template's expected namespace for OTel components.
Wait for the demo pods to become ready. The `otel-collector`, `prometheus`, `grafana`, `jaeger`, and `opensearch` pods should not be present with the values above.

[source,bash]
----
kubectl -n otel-demo get pods
----

== Verify telemetry reaches {stackstate-product-name}

Check that a demo service points at the Agent telemetry gateway:

[source,bash]
----
kubectl -n otel-demo describe deploy checkout | grep -A2 OTEL_COLLECTOR_NAME
----

Then open the {stackstate-product-name} UI and verify that OpenTelemetry services and service instances are visible. If no data appears, check the telemetry gateway troubleshooting guidance in xref:/setup/otel/telemetry-gateway.adoc[Telemetry gateway].

== Deploy the scaffolded StackPack

Create and test-deploy the generic StackPack:

[source,bash]
----
sts stackpack scaffold --name my-stackpack --display-name "My StackPack"
cd my-stackpack
STS_EXPERIMENTAL_STACKPACKS=1 sts stackpack test-deploy -d . --yes
----

The generic template is modeled on the OpenTelemetry Demo webshop. After it is installed, review the generated menu item, component overview, highlights, metrics, and monitor output in the {stackstate-product-name} UI.

== Clean up

Remove the demo workload when you are done:

[source,bash]
----
helm uninstall opentelemetry-demo --namespace otel-demo
kubectl delete namespace otel-demo
----
