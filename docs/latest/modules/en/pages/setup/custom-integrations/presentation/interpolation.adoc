== Interpolation

When defining Component Presentations, a couple of different types of expressions are used.  When selecting a value to xref:/setup/custom-integrations/presentation/projections.adoc[project] in a highlight field, a CEL expression can be used.  When binding the resulting time series of a PromQL query to components, a simpler variable interpolation is available.

[#cel_context]
=== CEL expression

CEL expressions are evaluated in an environment representing the component.  This is the case on most projections in highlight fields or overview columns, or in metrics.  The following variables are available

[cols="1,1,3"]
|===
|Variable               | Type                | Description

|name                   | string              | The name of the component
|typeName               | string              | The type of the component
|identifiers            | list<string>        | Identifiers of the component
|tags                   | map<string,string>  | Labels of the component, in key-value pairs
|properties             | map<string,dyn>     | Properties of the component
|healthState            | string              | One of the supported health states (CLEAR, DEVIATING, CRITICAL, UNKNOWN)
|lastUpdateTimestamp    | uint                | Timestamp of last update, in milliseconds since epoch
|===

==== CEL provisioning expression

When selecting an external component to provide configuration and status details, the `external` variable is available

[cols="2,1,2"]
|===
|Variable               | Type                | Description

|external.identifier    | string              | An identifier unique to the external component
|external.syncIdentifier| string              | The identifier of the component mapping
|external.tags          | string              | Tags contributed by the external component
|external.configuration | string              | Configuration of the (external) component
|===

=== PromQL expression

In a projection with a promql query, these variables can be used to create label selectors (e.g. `the_metric{namespace=${tags.namespace},ip=${properties.ip_address}}`)

[cols="1,1,3"]
|===
|Variable               | Type                | Description

|name                   | string              | The name of the component
|type                   | string              | The type of the component
|layer                  | string              | The layer of the component
|domain                 | string              | The domain of the component
|tags                   | map<string,string>  | Labels of the component, in key-value pairs
|properties             | map<string,dyn>     | Properties of the component
|===

=== Variable expression

When the available variables are dependent on a query, straightforward variable interpolation is possible.  This is the case when dealing with time series returned by a PromQL query.
