== New Features & Enhancements

* *SUSE Observability AI Investigation Agent:* Introduced a production-ready, autonomous diagnostic system designed to investigate and resolve Kubernetes incidents.
** *Autonomous Incident Investigation:* The agent uses a hypothesis-driven methodology (Orient → Hypothesize → Test → Conclude) to investigate infrastructure issues end-to-end, generating formal reports with root cause analysis and remediation steps.
** *Advanced Metric Analysis:* Implements a "hidden tool" pattern where Python statistical libraries analyze raw metric data (detecting trends, changepoints, and anomalies) and return compact, high-value summaries to the LLM, preventing context window bloat.
** *Model Context Protocol (MCP) Integration:* Built on a clean separation of concerns using a Go-based MCP tool server for data retrieval and a Python LangGraph runtime for orchestration and intelligence.
** *Multi-Provider Support:* Compatible with various LLM providers, including Anthropic, OpenAI, AWS Bedrock, and Ollama.
** *Rancher AI Alignment:* Architecturally aligned with the Rancher AI Agent (Liz), sharing common patterns for Helm charts, CRDs, and persistent checkpointing to ensure a unified SUSE AI experience.

* *Gateway API Support:* The SUSE Observability Helm chart now supports the Kubernetes Gateway API.
** *Global Configuration:* A single global flag allows users to switch between standard Ingress and the Gateway API seamlessly.
** *Simplified Setup:* Following Rancher-standard patterns, the configuration is minimized to require only the URL and TLS secret in most environments.
** *Documentation:* Updated installation guides now cover both Ingress and Gateway API implementation paths.

== Bug Fixes

* *Health State Aggregation Logic:* Improved the precedence logic for aggregated health states (e.g., at the Namespace or Cluster level). The *Clear* state now takes precedence over *Unknown*. 
** *Impact:* In scenarios where a component has many Unknown states and at least one Clear state (with no Critical or Deviating states), the overall health will now aggregate to *Clear* instead of *Unknown*.

=== Agent Bug Fixes

* *Agent Data Interaction:* Refactored the internal agent library to utilize the standardized MCP server for all StackState API interactions, ensuring more robust data validation and response encoding.
* *Version Compatibility:* This release is fully validated for use with **SUSE Observability Agent v1.2.28**.
