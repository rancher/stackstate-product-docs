= 追踪视角
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

追踪视角显示组件及其相关指标的跨度。
这使您能够直接在SUSE Observability中监控IT基础设施中应用程序的性能。

单击列表中的任何跨度以查看属于它的后代跨度。
展开时，跨度的时间线显示每个后代的开始和完成时间。
同一追踪中的父跨度默认是折叠的，但可以根据需要展开。 +
跨度根据其OpenTelemetry服务名称以不同颜色显示。
在检查追踪并查看其跨度列表时，您可以单击任何跨度以查看进一步的详细信息。

image::k8s/k8s-traces-perspective.png[追踪视角]

== 过滤追踪

追踪过滤器允许您根据跨度状态（错误、正常或未设置）、父类型（外部、内部或root）和持续时间来细化显示的追踪。

除了这些过滤器，追踪还与SUSE Observability UI底部时间线控件中选择的*时间窗口*匹配。
调整时间窗口以仅显示该时间段的追踪。

=== 过滤持续时间

如第二张截图所示，可以通过拖选直方图来选择跨度持续时间的区间。
缩小视图可以通过清除所有过滤器或刷选整个范围来实现。

== 跨度详细信息

在SUSE Observability中，xref:/use/views/k8s-view-structure.adoc[视图]显示特定Kubernetes资源的IT基础设施子集。
追踪视角显示与该资源相关的跨度及其后代。
由于后代跨度可以源自其他资源，因此可以从跨度详细信息中导航到它。
“父跨度”链接将展开视图以显示完整的追踪。

image::k8s/k8s-traces-perspective-span-details.png[后代跨度详细信息]

上面的两张图片通过展示一个主要负责从购物车创建订单的结账服务来说明这些概念。
您可以看到一个请求下订单的追踪及其跨度的示例。
一个后代跨度已被突出显示，其详细信息显示，包括与之相关的组件链接。

同样，刷选一个追踪将放大该追踪的特定时间段。
可以通过稍后点击来重置选择。

== 时间旅行

在使用追踪视角时，就像在其他视角中一样，您可以处于实时模式或xref:/use/stackstate-ui/k8sTs-timeline-time-travel.adoc#_time_travel[时间旅行到过去]。
