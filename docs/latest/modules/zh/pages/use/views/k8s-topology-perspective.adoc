== 概述

拓扑视角显示了您的IT环境中的组件及其关系。

image::k8s/k8s-topology-perspective.png[拓扑视角]

== 图例

点击屏幕右下角的图例按钮（在缩放控件正下方）以显示拓扑可视化中使用的图标和颜色的解释。

image::k8s/k8s-topology-perspective-legend.png[拓扑视角图例]

== 组件

拓扑视角显示了在选定的xref:/use/views/k8s-view-structure.adoc[视图]中过滤后的组件和关系。配置了一个或多个xref:/use/alerting/k8s-monitors.adoc[监视器]的组件将报告计算出的健康状态。

* 选择一个组件以在右侧面板详细信息选项卡中显示详细的组件信息 - *组件详细信息*。
* 将鼠标悬停在组件上以打开<<_component_context_menu,组件上下文菜单>>。

=== 组件上下文菜单

当您将鼠标指针悬停在组件上时，将显示组件上下文菜单。这将为您提供有关组件的信息，包括：

* 组件名称和类型
* 健康状态
* <<_actions,特定于组件的操作>>
* <<_shortcuts,特定于组件的快捷方式>>

image::k8s/k8s-component-contex-menu.png[组件上下文菜单]

=== 行动

操作可用于扩展拓扑选择，以显示所选组件的所有依赖关系。对于特定组件，可能还有其他操作，例如作为 StackPack 的一部分安装的组件操作。

在您选择一个组件时，右侧面板详细信息选项卡中包含可用的*操作*列表 - *组件详细信息*。组件上下文菜单中也列出了操作，当您将鼠标指针悬停在组件上时会显示该菜单。

=== 快捷键

快捷方式使您可以直接访问有关特定组件的详细信息：

* *打开组件视图* - 打开此组件的xref:use/views/k8s-metrics-perspective.adoc[组件指标]。组件视图为您提供了关于该组件及其直接邻居的一切重要信息的鸟瞰图，具体取决于您正在查看的组件类型。
* *探索组件* - 打开一个仅包含此组件的xref:/use/views/k8s-explore-views.adoc[探索视图]。探索视图允许您从各个角度调查单个组件，而无需调整视图过滤器。双击组件也能达到相同的效果。
* *显示属性* - 打开该组件的属性弹出窗口。这与在右侧面板详细信息选项卡中点击*显示所有属性*时的效果相同，当显示有关组件的详细信息时 - *组件详细信息*。

== 关系

关系显示了拓扑中组件之间的连接方式。它们由虚线或实线表示，并带有箭头，显示它们所连接组件之间的依赖方向。

* *直接关系* - 通过网络和任何其他形式的直接关系连接的组件。
* *间接关系* - 由于网络流量或其他依赖关系引起的组件之间的关系，不包括与当前拓扑选择之外的外部组件之间的连接。
* *分组关系* - 直接关系和间接关系的组合，连接组件组中的一个或多个组件。

选择一个关系以在右侧面板详细信息选项卡中打开有关它的详细信息。

== 可视化设置

在可视化器右上角的可视化设置菜单中，可以自定义拓扑视角中组件和关系的显示方式。 可视化设置与视图一起保存。

=== 网格选项

拓扑视角中的组件可以选择以网格形式组织，层按行分组，域按列可视化。网格默认开启。当没有选择网格选项时，不会显示网格。

* *按层组织* - 同一层的组件在拓扑可视化中放置在同一网格行中。禁用以从网格中去除行。
* *按域组织* - 同一域的组件在拓扑可视化中放置在同一网格列中。禁用以从网格中去除列。

例如，如果您有一个来自四个或五个不同来源的业务服务可视化，来源（域）在可视化中就不会那么重要。这可能是一个很好的情况，可以在关闭*按域组织*的情况下可视化拓扑。

=== 组件分组

大量组件的拓扑可视化可能会变得难以阅读。SUSE Observability可以在同一视图中对组件进行分组。分组将组件和关系的数量减少到视觉上更易于管理的程度。

分组默认启用，并遵循所选的<<_grid_options,网格选项>> - 组件必须在同一网格行/列中才能被分组在一起。

提供三种类型的分组，或者您可以选择不将组件分组在一起：

* *不分组* - 组件和关系没有以任何方式分组。
* *自动分组* - 分组设置会自动调整，以保持可视化的组件或组件组数量低于35。
* *按状态和类型分组* - 同一类型且健康状态相同的组件被分组在一起。如果其中一个分组组件的健康状态发生变化，不再与组件组的健康状态匹配，它将从组中弹出。如果同一类型的其他组件具有相同的健康状态，将创建一个新组。
* *按类型、状态和关系分组* - 组件被分组在一起，类似于*按状态和类型分组*，同时保持与组外组件的关系信息。组中的所有组件具有相同的源和目标连接。这很有用，因为仅通过状态和类型将组件分组可能会导致组中组件的关系信息丢失。

== 导航

=== 放大和缩小

拓扑可视化器的右下角有缩放按钮。*加号*按钮放大拓扑，*减号*按钮缩小。两个按钮之间是*适应屏幕*按钮，它会缩放以显示完整的拓扑结构。

=== 查找组件

您可以通过点击`CTRL` + `SHIFT` + `F`并输入组件名称的前几个字母来定位拓扑中的特定组件。或者，您可以选择拓扑可视化器右下角的*查找组件*放大镜图标。

查看xref:/use/stackstate-ui/k8sTs-keyboard-shortcuts.adoc[SUSE Observability 键盘快捷键]的完整列表。

=== 显示根本原因

如果有监视器配置的组件位于视图之外但可能影响视图中的组件，拓扑视角将显示视图中所有组件的健康状态。

* *不显示根本原因* - 不显示当前拓扑过滤器所显示组件的根本原因。
* *仅显示根本原因* - 仅显示当前拓扑过滤器所显示的具有`CRITICAL`或`DEVIATING`传播健康的组件的根本原因。如果一个组件直接依赖于至少一个不可见组件并导致根本原因，则间接关系会被可视化。
* *显示完整根本原因树* - 显示当前拓扑过滤器所显示的具有`CRITICAL`或`DEVIATING`传播健康的组件到其根本原因的所有路径。

image::k8s/k8s-show-root-cause.png[根本原因]

== 列表模式

拓扑可视化中的组件也可以以列表形式显示，而不是图形形式：

image::k8s/k8s-topology-perspective-list-mode.png[过滤（列表格式）]

=== 导出为 CSV

在列表模式下，组件列表可以导出为 CSV 文件。CSV 文件包含每个组件在视图中的 `name`、`state`、`type` 和 `updated` 详细信息。

. 从拓扑视角，点击右上角的 *列表模式* 图标以在列表模式下打开拓扑。
. 点击页面顶部的 *下载为 CSV*。
 ** 组件列表将作为名为 `<view_name>.csv` 的 CSV 文件下载。
