= 探索视图
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

== 概述

一般来说，SUSE Observability 中的 xref:/use/views/k8s-view-structure.adoc[视图] 概念允许您监控您之前保存的 IT 领域。但是，当您需要调查特定视图的子集时，您通常不想失去该视图的范围。这就是 *探索视图* 发挥作用的地方。

为了保持特定视图（例如 `my view`）的范围不变，应用于拓扑元素或元素选择（例如组件、关系、组）的所有调查操作将自动在您开始的视图下以临时探索视图的形式打开（例如 `my view / explore`）。

将在探索视图中自动打开的 *调查操作* 示例：

* 在 xref:/use/views/k8s-topology-perspective.adoc[拓扑视角] 中，单击了组件上的 *+ 按钮*，以显示该组件的隐藏邻居。
* 在组件上执行了 *快速操作*。
* 在 xref:/use/views/k8s-topology-perspective.adoc[拓扑视角] 中，*双击* 了一个拓扑元素（组件、关系、组）以进行调查。
* 在拓扑元素（组件、关系、组）上单击了 *'探索...'* 链接或按钮以进行调查。

[TIP]
====
从现有探索视图执行的调查操作将 *不会* 打开新的探索视图——它们将改变您所在的探索视图的范围。
====


== 处理

当自动创建探索视图时，其原始范围也会被定义。如果您想在对其进行一些更改后将探索视图重置为其原始范围，这将很有帮助。

尽管探索视图旨在是临时的，但如果您想监控或稍后查看它，可以将其保存为 xref:/use/views/k8s-custom-views.adoc[自定义视图]。

如果您不想保存探索视图，只需远离它或通过使用顶部导航栏上的面包屑返回到您开始的视图。
