== 概述

SUSE Observability 对 Kubernetes 及其组件有深入的了解。在您的集群中安装 SUSE Observability 代理后，它将自动检测并可视化您的 Kubernetes 应用程序的拓扑结构。这包括构成您应用程序的 Kubernetes 资源，例如部署、Pod、服务和 Ingress。它还将自动检测并可视化构成您集群的 Kubernetes 基础设施的拓扑结构，例如节点、名称空间和持久卷。

SUSE Observability 为以下 Kubernetes 原生资源提供了专门的概述和亮点页面：

image::k8s/k8s-menu.png[Kubernetes 视图]

所有其他 Kubernetes 资源都在拓扑视图中被识别和可视化。

== 概述页面

概述页面提供了您环境中特定 Kubernetes 资源的高层次概述。这些概述突出了有关资源的最重要信息，包括其健康状况、年龄、名称空间、集群等。

image::k8s/k8s-service-overview.png[概述页面]

从概述页面，您可以点击其中一个 Kubernetes 资源以导航到该资源的亮点页面。

== 亮点页面

亮点页面显示有关特定 Kubernetes 资源的最重要信息的概述。在这里，您可以看到资源的健康状况、在该资源上活动的监控器以及在该资源上发生的事件。我们还展示了该资源的关键指标。

image::k8s/k8s-pod-highlights.png[亮点页面]
