== 概述

SUSE Observability 通过监控实现了最佳实践。SUSE Observability 不仅监控指标、事件或日志，还可以监控拓扑和元数据，以检测常见问题，确保符合行业标准。
将监控与我们独特的故障排除智能相结合，SUSE Observability 能够快速检测相关问题，并提供修复建议。这种主动的方法降低了未检测到问题的风险，并有助于维护健康和强大的 Kubernetes 环境。
监控配有针对触发警报的资源及其监控条件的修复指南，帮助用户进行故障排除。

== 监视结果

监控结果显示在两个地方：

. 在所有资源的高亮页面上，立即查看发生了什么以及如何修复。
. 在右侧的侧窗格中，如果您在拓扑中选择一个元素，可以快速了解该组件的问题。

== 修复指南

修复指南显示在监控状态抽屉中，当您通过点击资源高亮视图中的消息打开监控时会显示。
