当通知通道无法发送通知时，通知概览将显示错误状态。检查错误：

. 点击通知
. 页面顶部显示一个横幅，概述错误信息
. 向下滚动到通道，点击受影响通道的“错误”链接以打开错误详情。

== 常见错误

=== Slack

Slack 错误通常包含一个错误代码：

* `token_revoked`、`token_expired` 或 `missing_scope`：这些错误代码都表示 SUSE Observability 在向 Slack 发送消息时的授权问题。要解决这些问题，请重新创建通道以重新授权 SUSE Observability 与 Slack，从而授予所有所需的权限。
* 有关其他错误代码，请参见 https://api.slack.com/automation/cli/errors[Slack API 错误代码]。

=== HTTP

大多数通道，特别是 webhook，期望返回状态为 200 OK 的 HTTP 响应。其他响应被视为错误。HTTP 错误包含一个通常较短的消息和接收到的状态代码，而不是 200 OK。

验证任何配置的 URL 和词元是否正确，并且在 webhook 的情况下，确保 POST 请求被正确处理并返回 200 OK 响应。

=== 其他

如果错误类型为“其他”，则 SUSE Observability 很可能无法连接到外部服务或 webhook。验证 URL 是否正确、可解析并且可以被 SUSE Observability 访问。
