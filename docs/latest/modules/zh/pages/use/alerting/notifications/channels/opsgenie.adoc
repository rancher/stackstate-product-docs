== 配置 Opsgenie 通知

要向 Opsgenie 发送通知，请按照以下步骤操作：

. <<_create_integration,在 OpsGenie 中创建集成>>
. <<_create_and_test_the_channel,创建频道>>

=== 创建集成

为了将 SUSE Observability 与 OpsGenie 集成，需要创建一个全局的 "API 密钥"。 每个通知的响应者可以不同，并将在 SUSE Observability 中进行配置。

在 OpsGenie 中，转到 "设置 \-> 集成"。

* 点击 "添加集成" 按钮
* 搜索 "API" 集成类型。
* 为此集成选择一个名称，但 _请勿选择分配团队_。

当集成创建后，它将拥有一个 "API 密钥"。

=== 创建并测试频道

image::k8s/notifications-opsgenie-channel-configuration.png[创建频道]

选择适合您 OpsGenie 账户的区域并输入 "API 密钥"。 为通知选择响应者；_用户_、_团队_、_时间表_ 和 _升级_ 可用于此。

点击 "添加频道" 按钮。这将把频道添加到右侧的频道列表中。它将显示一个“测试”按钮。按下它将在 OpsGenie 中生成一个测试警报。

== OpsGenie警报用于通知

当打开通知时，会创建一个新的OpsGenie警报。此消息将在更改时更新，通常仅在健康状态变化时。当通知关闭时，OpsGenie警报也会再次关闭。 警报中的备注包含关闭它的原因。
