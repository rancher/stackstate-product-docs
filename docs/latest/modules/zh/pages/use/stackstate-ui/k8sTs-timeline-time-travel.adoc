== 概述

SUSE Observability UI 底部的时间线允许您回到过去特定时刻的拓扑状态。然后，您可以浏览所选拓扑快照的所有遥测数据。时间线中的健康和事件图表概述了在所选遥测时间间隔内拓扑的状态。

image::k8s/k8s-timeline.png[时间线]

== 时间线

=== 遥测时间间隔

遥测时间间隔指定了在 SUSE Observability 视角中可用事件、指标和追踪的时间窗口。它在时间线上从左到右运行。

image::v51_telemetry_interval.png[遥测时间间隔]

所选的遥测时间间隔可以是相对的（实时模式），也可以设置为自定义遥测时间间隔（时间旅行模式）。默认情况下，遥测时间间隔设置为相对遥测时间间隔 - 在实时模式下显示过去一小时的遥测数据。您可以放大/缩小或设置自定义遥测时间间隔，以查看特定时间点的遥测数据。

==== 设置遥测时间间隔

[NOTE]
====

* 遥测时间间隔最长可为 6 个月。
* 当为遥测时间间隔设置自定义遥测时间间隔时，SUSE Observability 将暂停 <<_topology_time,拓扑时间>> 并进入 <<_time_travel,时间旅行模式>>。
====


遥测时间间隔可以通过以下方式设置：

* *放大*

image::v51_timeline_click_drag.png[在时间线上单击并拖动以在您的选择上设置自定义遥测时间间隔]

* *缩小*

image::v51_telemetry_interval_zoom_out.png[点击放大镜以将遥测时间间隔的大小加倍]

* *使用遥测时间间隔跳转箭头*

image::v51_telemetry_interval_jumper.png[点击时间跳转箭头以在时间上向后或向前移动遥测时间间隔]

* *设置相对或自定义的遥测时间间隔*

image::v51_timeline_telemetry_interval.png[使用弹出窗口 "设置遥测时间间隔" 来指定遥测时间间隔]

=== 拓扑时间

SUSE Observability中的拓扑基于您环境的快照，该快照是在该时刻观察到的。此快照的时间点由拓扑时间指定。默认情况下，SUSE Observability处于实时模式，拓扑时间设置为当前时间。您可以通过选择自定义拓扑时间来<<_time_travel,时间旅行>>到拓扑的先前状态。这有助于您在某个时刻调查问题。即使一个pod已经消失，您仍然可以看到它是如何连接的、它的日志、事件、相关资源、事件等。

image::v51_topology_time.png[拓扑时间]

在时间轴上，所选的拓扑时间由播放头指示——一条黑线，顶部显示当前拓扑时间。它也在时间轴顶部的*拓扑时间*框中指定。

==== 设置拓扑时间

拓扑时间可以通过以下方式设置：

* *点击时间轴*

image::v51_topology_time_timeline.png[在时间轴上点击任意位置以将拓扑时间设置为该时刻]

* *使用拓扑时间跳转箭头*

image::v51_topology_time_jumper.png[点击拓扑时间跳跃箭头以将拓扑时间向前或向后移动到下一组事件]

* *设置自定义拓扑时间*

image::v51_topology_time_popup.png[使用弹出窗口"设置拓扑时间"来指定拓扑时间]

* *点击时间戳*

image::v51_topology_time_timestamp.png[点击时间戳以跳转到该特定拓扑时间]

==== 拓扑时间超出遥测时间间隔

如果所选的拓扑时间超出了当前选定的<<_telemetry_interval,遥测时间间隔>>，将显示消息"拓扑时间超出当前遥测时间间隔"，并且时间轴顶部的*拓扑时间*框将被突出显示为黑色。由于时间轴从左到右显示遥测时间间隔，指示当前拓扑时间的播放头在时间轴上将不可见。

image::v51_topology_time_outside_telemetry_interval.png[拓扑时间超出遥测时间间隔]

您仍然可以按预期浏览拓扑和遥测：

* 在xref:/use/views/k8s-topology-perspective.adoc[拓扑视角]中，所选拓扑时间的拓扑状态被可视化。
* 在所有视角中，显示在所选遥测时间间隔内生成的遥测，并与在所选拓扑时间存在的拓扑元素相关。

==== 实时模式

要停止时间旅行并将拓扑时间返回到实时模式，请点击屏幕顶部的*返回实时*或*返回实时*。

image::v51_timeline_go_live.png[返回实时]

=== 健康

在所选遥测时间间隔内，视图的健康状态以颜色显示在时间轴*健康*线上。

对于单个资源，健康状态将随时间显示，在概览中显示为灰色线。

image::v51_timeline_no_health_state.png[健康状态不可用]

=== 事件活动

时间轴中的*事件*线显示每个时间点生成的事件数量的柱状图。这帮助您看到过去充满活动的时刻。请注意，仅显示在所选的<<_topology_time,拓扑时间>>存在的拓扑元素生成的事件。

要放大感兴趣的事件条，请单击并拖动以在时间轴上选择一个较小的遥测时间间隔。

image::v51_timeline_click_drag.png[单击并拖动以选择遥测时间间隔]

[NOTE]
====
在时间轴上单击一次将播放头移动到此时刻，从而实现时间旅行，回到所选的<<_topology_time,拓扑时间>>的状态。现在仅显示在新选择的拓扑时间存在的拓扑元素生成的事件。
====


== 时间旅行

在每个SUSE Observability 视角中，您可以处于实时模式或过去模式。在实时模式下，SUSE Observability 将不断轮询新数据。当您通过拓扑或遥测进行时间旅行时，您实际上是在处理基础设施的快照。可用的数据基于两个选择：

* <<_topology_time,拓扑时间>> - 您想要获取Kubernetes资源快照的特定时刻。
* <<_telemetry_interval,遥测时间间隔>> - 您想查看遥测和追踪的时间范围。

要停止时间旅行并返回实时模式，请单击屏幕顶部的*返回实时*或*返回实时*。

image::v51_timeline_go_live.png[返回实时]
