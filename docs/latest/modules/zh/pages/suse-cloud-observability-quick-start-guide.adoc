== 概述

在从云服务提供商市场购买 SUSE Cloud Observability 后，您的 SUSE Cloud Observability 环境将被配置。

您将收到来自 SUSE Cloud Observability 的电子邮件，其中包含所需的登录详细信息和指向您环境的链接。本快速入门指南将帮助您开始使用并将自己的数据导入到您的 SUSE Cloud Observability 部署中。

== 入门 - 第一步

. 设置密码
. 登录到您的 SUSE Cloud Observability 实例
. 将 SUSE Observability 代理安装到您的集群中
. 更改您的个人账户信息

=== 设置密码

来自 SUSE Cloud Observability 的电子邮件包含一个唯一链接，允许您在账户上设置初始密码。 在您可以登录并配置可观测性环境之前，必须执行此操作。

=== 访问您的 SUSE Observability 环境

通过点击您从 SUSE Cloud Observability 收到的电子邮件中的唯一链接登录到您的 SUSE Cloud Observability 环境。 输入您的密码将带您进入配置屏幕，以便添加集群。

ifdef::feature-agent-install-cluster[]
=== 将 SUSE Observability 代理安装到您的集群中

SUSE Cloud Observability 使用 StackPacks 来简化配置下游集群并将数据导入可观测性环境。

最初，您应该被带到以下屏幕，要到达这里，在 SUSE Observability 用户界面中，通过点击屏幕左上角打开主菜单，然后转到 `StackPacks` > `Kubernetes`。

image::integrating_first_cluster_stackpacks.png[SUSE Cloud Observability - StackPacks]

一旦您进入 Kubernetes StackPack 屏幕，向可观测性环境添加集群非常简单。

输入您希望观察的集群名称，它不必与 'kubeconfig' 中使用的集群名称匹配。

Kubernetes 集群名称必须以小写字母或数字开头和结尾，并且只能包含小写字母、数字、点和破折号（. -）

输入名称并单击`Install`按钮。 这将带您到以下屏幕，集群标记为“等待数据”。

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - 添加集群]

单击集群名称，这将展开此部分并显示一系列信息，包括先决条件和可以用于将现有集群添加到您的可观测性环境的命令。

查看信息并确保您对Kubernetes环境具有正确的权限，确保您正在运行受支持的Kubernetes版本。

这些命令是您可观测性部署的唯一命令，包括所需的API密钥和URL。 选择适合您集群的命令，包含EKS、RKE、通用Kubernetes集群等多个部分。

这些命令将安装SUSE Cloud Observability代理并将集群连接到您的SUSE Cloud Observability环境。

您可以直接从UI使用这些命令，或者可以按照xref:/k8s-quick-start-guide.adoc[快速入门指南]中提供的命令将其部署到下游集群。

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - 安装后]

连接集群后，SUSE Cloud Observability UI中应显示一个绿色勾号。

此时，您可以开始探索您的数据。
endif::feature-agent-install-cluster[]

=== 更改您的个人账户信息

您电子邮件中的第4步是更新您的个人信息。 单击电子邮件中的唯一链接，并根据需要添加您的基本个人信息。 如果需要，您还可以从此部分设置2FA身份验证。

=== 探索您的数据

要开始探索您的数据，请通过单击屏幕左上角打开主菜单，然后转到`Kubernetes`以显示可观察项目的列表。

image::accessing_views_1.png[SUSE Cloud Observability - 探索您的数据]

从基础设施部分选择“集群”，这将显示监控集群的列表，选择您的集群以显示众多内置视图之一。

image::accessing_views_2.png[SUSE Cloud Observability - 集群视图]

此时，您可以开始探索集群的数据或添加更多集群以收集数据。

有关如何使用 SUSE Cloud Observability 的更多信息，包括创建自定义视图，请参阅标准文档。

=== SUSE Cloud Observability 的限制

请注意，SUSE Cloud Observability 并未提供开箱即用的 RED 信号（速率、错误和持续时间），仅显示速率信号。 此功能适用于 SUSE Rancher Prime。
需要这些信号以获得完整可观测性解决方案的客户应联系 SUSE 讨论 SUSE Rancher Prime，或者可以使用 OpenTelemetry 收集器收集这些数据。

== 附加信息

有关更多信息，包括您的平台的先决条件和支持的 Kubernetes 版本，请参阅 xref:/k8s-quick-start-guide.adoc[快速入门]。
