= 配置 SUSE Observability Kubernetes Agent 以代理连接
:revdate: 2025-07-10
:page-revdate: {revdate}

SUSE Observability Kubernetes Agent 允许您为其发起的连接配置 HTTP 或 HTTPS 代理设置。

== 与 SUSE Observability 通信的代理

要配置 SUSE Observability Kubernetes Agent 以代理连接到 SUSE Observability 后端，您可以使用 Helm 配置。

=== Helm 配置

==== 通过 `values.yaml` 文件

. 打开您的 Helm 图表 `values.yaml` 文件。
. 找到 `global.proxy.url` 配置并指定代理 URL：
+
[,yaml]
----
 global:
   proxy:
     url: "https://proxy.example.com:8080"
----

. 可选地，如果代理没有签名证书，请通过将 `global.skipSslValidation` 设置为 `true` 来禁用 SSL 验证：
+
[,yaml]
----
 global:
   skipSslValidation: true
----

==== 通过命令行标志

. 在安装 Helm 图表时，使用 `--set` 标志指定代理 URL：
+
[,bash]
----
 helm install suse-observability-agent suse-observability/suse-observability-agent --set global.proxy.url="https://proxy.example.com:8080"
----

. 要通过命令行禁用 SSL 验证，请使用：
+
[,bash]
----
 helm install suse-observability-agent suse-observability/suse-observability-agent --set global.skipSslValidation=true
----
