== ComponentPresentation

用户界面展示模型的核心概念是 *组件展示*。

一个 `ComponentPresentation` 定义了一个或多个匹配组件在用户界面中的展示方式，具体取决于活动上下文。
它专注于 *展示关注点*，例如命名、图标、表格、概述和高亮页面、过滤器以及菜单项的位置。

从高层次来看，组件展示：

* 使用绑定（一个简单的 STQL 查询）选择组件。
* 在特定模式下应用（例如，可观测性）。
* 定义一个或多个展示方面（例如概述表或高亮）。
* 可以部分定义展示属性，并与其他匹配定义组合。

多个组件展示可以应用于同一组件。在这种情况下，它们的定义会结合在一起以确定最终展示。
