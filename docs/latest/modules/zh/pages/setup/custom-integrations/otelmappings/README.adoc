{stackstate-product-name}可以自动从OpenTelemetry (OTel) 遥测数据中推导应用程序和基础设施拓扑。
这是通过OpenTelemetry拓扑映射实现的，这些映射以声明方式将跟踪和指标信号转换为拓扑组件（顶点）和关系（边）。

== 概述

从高层次来看，OpenTelemetry的拓扑生成工作如下：

* OpenTelemetry遥测（指标和/或跟踪）由收集器接收。
* OTel收集器使用拓扑连接器，根据配置的映射评估遥测数据。
* 当映射条件与遥测数据匹配时，生成组件和/或关系。
* 这些拓扑对象被`sync`服务使用，并合并到拓扑图中。
* 映射是声明式的，基于YAML，并在遥测数据流经系统时持续评估。

当前支持的信号：

* 指标
* 跟踪

[NOTE]
====
基于日志的拓扑生成尚不支持。
====

有关组件和关系映射的更多信息，请参见xref:/setup/custom-integrations/otelmappings/concepts.adoc[concepts]页面。
