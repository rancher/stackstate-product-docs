== Descripción general

Después de adquirir SUSE Cloud Observability en el Marketplace del Proveedor de Nube, se provisiona tu entorno de SUSE Cloud Observability.

Recibirás un correo electrónico de SUSE Cloud Observability con los detalles de inicio de sesión requeridos y enlaces a tu entorno. Esta guía de inicio rápido te ayudará a comenzar y a introducir tus propios datos en tu ampliación de SUSE Cloud Observability.

== Comenzando - Primeros Pasos

. Configura una contraseña
. Inicia sesión en tu instancia de SUSE Cloud Observability
. Instala el agente de SUSE Observability en tu clúster
. Cambia los detalles de tu cuenta personal

=== Configura una contraseña

El correo electrónico de SUSE Cloud Observability contiene un enlace único que te permite establecer tu contraseña inicial en la cuenta.  Esto debe realizarse antes de que puedas iniciar sesión y configurar el entorno de observabilidad.

=== Accediendo a tu Entorno de SUSE Observability

Inicia sesión en tu entorno de SUSE Cloud Observability haciendo clic en el enlace único en el correo electrónico que recibiste de SUSE Cloud Observability.  Introducir tu contraseña te llevará a la pantalla de configuración para añadir clústeres.

ifdef::feature-agent-install-cluster[]
=== Instala el agente de SUSE Observability en tu clúster

SUSE Cloud Observability utiliza StackPacks para facilitar la configuración de tus clústeres en sentido descendente y obtener datos en el entorno de observabilidad.

Inicialmente deberías ser llevado a la siguiente pantalla, para llegar aquí, en la interfaz de SUSE Observability, abre el menú principal haciendo clic en la parte superior izquierda de la pantalla y ve a `StackPacks` > `Kubernetes`.

image::integrating_first_cluster_stackpacks.png[SUSE Cloud Observability - StackPacks]

Una vez que estés en la pantalla de Kubernetes StackPack, es muy sencillo añadir un clúster al entorno de observabilidad.

Introduce un nombre para el clúster que deseas observar, no tiene que coincidir con el nombre del clúster utilizado en 'kubeconfig'.

El nombre del clúster de Kubernetes debe comenzar y terminar con una letra minúscula o un dígito y puede consistir únicamente en letras minúsculas, dígitos, puntos y guiones (. -)

Introduce el nombre y haz clic en el botón `Install`.  Esto debería llevarte a la siguiente pantalla con el clúster marcado como 'Esperando datos'

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - Añadiendo un Clúster]

Haz clic en el nombre del clúster, esto expandirá esta sección y revelará una serie de información incluyendo requisitos previos y comandos que se pueden usar en tus clústeres existentes para añadirlos a tu entorno de observabilidad.

Revisa la información y asegúrate de tener los permisos correctos para tu entorno de Kubernetes, asegúrate de estar ejecutando una versión compatible de Kubernetes.

Estos comandos son únicos para tu ampliación de observabilidad e incluyen las claves API y URLs requeridas.  Selecciona los comandos apropiados para tu clúster, hay secciones para EKS, RKE, clústeres genéricos de Kubernetes y muchos más.

Estos comandos instalarán los agentes de SUSE Cloud Observability y conectarán el clúster a tu entorno de SUSE Cloud Observability.

Puedes usar los comandos directamente desde la interfaz de usuario o puedes desplegar en tus clústeres en sentido descendente siguiendo los comandos proporcionados en la sección 'Desplegar el Agente StackState y el Agente del Clúster' en la xref:/k8s-quick-start-guide.adoc[guía de inicio rápido].

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - Post Instalación]

Después de que el clúster se haya conectado, debería haber una marca verde en la interfaz de usuario de SUSE Cloud Observability.

En este punto, puedes comenzar a explorar tus datos.
endif::feature-agent-install-cluster[]

=== Cambia los detalles de tu cuenta personal

El paso 4 de tu correo electrónico es actualizar tu información personal.  Haz clic en el enlace único del correo electrónico y añade tus datos personales básicos según sea necesario.  También puedes configurar la autenticación 2FA desde esta sección si es necesario.

=== Explora tus datos

Para comenzar a explorar tus datos, abre el menú principal haciendo clic en la parte superior izquierda de la pantalla y ve a `Kubernetes` para revelar una lista de elementos observables.

image::accessing_views_1.png[SUSE Cloud Observability - Explorando Tus Datos]

Selecciona 'Clústeres' de la sección de infraestructura, que debería mostrar una lista de clústeres monitorizados; selecciona tu clúster para revelar una de las muchas vistas integradas.

image::accessing_views_2.png[SUSE Cloud Observability - Vista del Clúster]

En este punto, puedes empezar a explorar los datos de tu clúster o añadir más clústeres de los cuales recopilar datos.

Para más información sobre cómo utilizar SUSE Cloud Observability, incluyendo la creación de vistas personalizadas, por favor consulta la documentación estándar.

=== Limitaciones de SUSE Cloud Observability

Ten en cuenta que SUSE Cloud Observability no proporciona señales RED (Tasa, Errores y Duración) de forma predeterminada; solo se muestra la señal de Tasa.  Esta función está disponible con SUSE Rancher Prime.
Los clientes que necesiten estas señales para tener una solución de Observabilidad completa deben contactar con SUSE para discutir SUSE Rancher Prime; alternativamente, estos datos pueden ser recopilados utilizando los recolectores de OpenTelemetry.
