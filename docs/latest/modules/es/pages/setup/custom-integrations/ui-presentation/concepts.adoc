== Presentación de Componentes

El concepto central del modelo de presentación de la interfaz de usuario es la *Presentación de Componentes*.

Una `ComponentPresentation` define cómo se presentan uno o más componentes coincidentes en la interfaz de usuario, dependiendo del contexto activo.
Se centra exclusivamente en *preocupaciones de presentación*, como nombres, iconos, tablas, páginas de resumen y destacados, filtros y colocación de elementos de menú.

A un alto nivel, una Presentación de Componentes:

* Selecciona componentes utilizando un enlace (una consulta STQL simple).
* Se aplica dentro de un modo específico (por ejemplo, observabilidad).
* Define uno o más aspectos de presentación (como tablas de resumen o destacados).
* Puede definir parcialmente propiedades de presentación y componer con otras definiciones coincidentes.

Múltiples Presentaciones de Componentes pueden aplicarse al mismo componente. En tales casos, sus definiciones se combinan para determinar la presentación final.
