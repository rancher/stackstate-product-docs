== Descripción general

Kubernetes mantiene registros exhaustivos de la actividad del clúster y de la salida de la aplicación. El registro proporciona información valiosa sobre el rendimiento de los contenedores, los nodos y Kubernetes en sí. Registrando meticulosamente todo, desde Pods hasta ReplicaSets, permite rastrear problemas hasta su origen.

== Accediendo a los registros de pods

Los errores y advertencias en los registros son indicadores de problemas potenciales en tu aplicación, contenedores o el propio pod. Para explorar los registros, cuando estés en la pantalla del pod, haz clic en el botón *Mostrar registros*.

== Usando los registros de pods

Puedes usar la búsqueda para buscar errores o advertencias específicas en los registros. Los registros se pueden filtrar por contenedor. Puedes seleccionar un intervalo de tiempo deseado para mostrar los registros y un intervalo de actualización para la frecuencia con la que te gustaría que se actualicen los registros.

El histograma de registros visualiza el número de líneas de registro dentro de un intervalo de tiempo seleccionado y te permite hacer zoom en un intervalo particular haciendo clic y arrastrando.

Utiliza estas opciones para mostrar u ocultar información adicional para cada línea de registro:

* *Mostrar hora* para mostrar las marcas de tiempo
* *Mostrar contenedores* para mostrar el nombre del contenedor al que pertenece la línea de registro
* *Ajustar líneas* para habilitar el ajuste de las líneas largas
* *Más nuevo primero/Más antiguo primero* para cambiar el orden de clasificación de los registros
