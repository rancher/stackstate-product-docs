== Configurar notificaciones de Opsgenie

Para enviar notificaciones a Opsgenie, sigue estos pasos:

. <<_create_integration,Crear una integración en OpsGenie>>
. <<_create_and_test_the_channel,Crear el canal>>

=== Crear integración

Para integrar SUSE Observability con OpsGenie, es necesario crear una "clave API" global.  Los respondedores de una notificación pueden ser diferentes para cada notificación y se configurarán en SUSE Observability.

En Opsgenie, ve a "Configuración \-> Integraciones".

* Haz clic en el botón "Añadir integración"
* Busca el tipo de integración "API".
* Elige un nombre para esta integración, pero _no selecciones un equipo asignado_.

Cuando se haya creado la integración, tendrá una "clave API".

=== Crea y prueba el canal

image::k8s/notifications-opsgenie-channel-configuration.png[Crear el canal]

Selecciona la región apropiada para tu cuenta de OpsGenie e introduce la "clave API".  Elige respondedores para la notificación; _usuarios_, _equipos_, _horarios_ y _escalaciones_ están disponibles para esto.

Haz clic en el botón "Añadir canal". Esto añade el canal a la lista de canales a la derecha. Mostrará un botón de "Prueba". Presionarlo generará una alerta de prueba en OpsGenie.

== Alertas de OpsGenie para notificaciones

Cuando se abre una notificación, se crea una nueva alerta de OpsGenie. Este mensaje se actualizará para cambios, generalmente solo cuando cambie el estado de salud. Cuando se cierra la notificación, la alerta de OpsGenie se cierra de nuevo.  Una nota en la alerta contiene la razón para cerrarla.
