== Accédez aux journaux

Vous pouvez explorer les journaux de conteneurs de tout environnement Kubernetes configuré avec l'envoi de journaux via l'interface utilisateur de SUSE Observability.
Cela peut se faire en descendant jusqu'au niveau `pod` via l'un des paradigmes fournis (Services, Déploiements, Stateful Sets, Daemon Sets, etc.) dans le menu Kubernetes à gauche

image::k8s/k8s-menu.png[Menu des Paradigmes Kubernetes]

Pour des raisons de simplicité et de complétude, l'exemple utilise directement le paradigme `Pods`, et un pod a été choisi contenant plusieurs conteneurs, afin de pouvoir distinguer entre les journaux de Pod et les journaux de Conteneur.  La vue Pod aura plusieurs points d'entrée liés aux journaux.

== Affichage des journaux

image::k8s/k8s-pod-view-node-agent.png[Vue du Pod Kubernetes]

Pour voir tous les journaux associés à ce Pod, cliquez sur "Voir tout", entouré en rouge.  Pour filtrer par type d'entrée de journal (Erreurs, Avertissements, Autres), sélectionnez l'un des éléments entourés en bleu.  L'histogramme affichant la proportionnalité entre les types d'entrées de journal est entouré en vert.

=== Le tiroir des journaux

Le tiroir des journaux facilite une expérience visuellement agréable pour naviguer dans les fichiers journaux d'une manière qui améliore l'expérience de dépannage.

Pour des raisons de simplicité, l'exemple ne traite que des sélections non filtrées, qui afficheront toutes les lignes de journal, de tous les conteneurs, de tout type d'entrée de journal.  Sélectionner `View all` ouvre le tiroir des journaux sans filtrage appliqué :

image::k8s/k8s-pod-view-log-drawer-no-filter.png[Vue des journaux Kubernetes sans filtres]

À partir de ce point, on pourrait commencer à se rapprocher du problème en appliquant des options aux filtres fournis (correspondance de chaîne de recherche, gravité, conteneur, intervalle de fenêtre de journal).  Plusieurs options sont disponibles pour le tri et les préférences d'affichage juste en dessous de l'histogramme dans le tiroir des journaux.

* La correspondance de chaîne de recherche prendra un terme de recherche en entrée et filtrera toute ligne de journal qui ne le contient pas.
* Les options de gravité sont `Any`, `Error`, `Warning` et `Other`. Sélectionner autre chose que `Any` exclura toutes les lignes de journal qui ne correspondent pas à la sélection.
* Les options de conteneur incluront `Any`, ainsi qu'une liste de conteneurs dans le Pod.  Sélectionner autre chose que `Any` restreindra la sortie à celle produite par le conteneur sélectionné.
* Les options d'intervalle de fenêtre de journal seront une interface de sélection de date-heure qui vous permettra de spécifier un intervalle de `from` et `to`.  Modifier cela par rapport à la valeur par défaut exclura toutes les entrées de journal qui tombent en dehors de la période sélectionnée.

À titre d'exemple, les quatre peuvent être vus en action ci-dessous :

image::k8s/k8s-pod-view-log-drawer-with-filters.png[Vue des journaux Kubernetes avec filtres]

L'image ci-dessus a les filtres suivants appliqués :

* Terme de recherche : `wrong type in json response`
* Gravité : `Warning`
* Conteneur : `node-agent`
* Intervalle de fenêtre de journal : Entre `02:00am` et `03:00am` du jour en cours.
