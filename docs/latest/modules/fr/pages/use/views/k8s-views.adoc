== Présentation

SUSE Observability a une connaissance approfondie de Kubernetes et de ses composants. Après l'installation de l'agent SUSE Observability dans votre cluster, il détectera et visualisera automatiquement la topologie de vos applications Kubernetes. Cela inclut les ressources Kubernetes qui composent votre application, telles que les déploiements, les pods, les services et l'ingress. Il détectera également et visualisera automatiquement la topologie de votre infrastructure Kubernetes qui compose votre cluster, tels que les nœuds, les espaces de noms et les volumes persistants.

SUSE Observability a des pages d'aperçu et de mise en évidence dédiées pour les ressources natives Kubernetes suivantes :

image::k8s/k8s-menu.png[Vues Kubernetes]

Toutes les autres ressources Kubernetes sont reconnues et visualisées dans les vues de topologie.

== Pages d'aperçu :

Les pages d'aperçu fournissent une vue d'ensemble de haut niveau des ressources Kubernetes spécifiques dans votre environnement. Ces aperçus mettent en évidence les informations les plus importantes concernant la ressource, y compris son état, son âge, son espace de noms, son cluster et plus encore.

image::k8s/k8s-service-overview.png[Pages d'aperçu]

Depuis une page d'aperçu, vous pouvez cliquer sur l'une des ressources Kubernetes pour naviguer vers la page de mise en évidence de cette ressource.

== Pages de mise en évidence :

La page des points forts présente un aperçu des informations les plus importantes concernant une ressource Kubernetes spécifique. Ici, vous pouvez voir l'état de la ressource, les moniteurs actifs sur la ressource et les événements qui se sont produits sur la ressource. Nous affichons également les indicateurs clés pour la ressource.

image::k8s/k8s-pod-highlights.png[Pages de mise en évidence]
