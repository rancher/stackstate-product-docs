== Présentation

Une vue dans SUSE Observability vous permet de surveiller et d'inspecter un sous-ensemble de votre environnement informatique. La structure d'une vue est conçue pour filtrer et visualiser les données de ce sous-ensemble (vue) de manière efficace.

=== Filtres

Le menu *Filtres* dans le coin supérieur droit de l'interface utilisateur de la vue vous permet de filtrer les composants (topologie), les événements et les traces affichés dans une vue. Une fois appliqués, les filtres affecteront le contenu de toutes les perspectives dans une vue.

* xref:/use/views/k8s-filters.adoc[Filtres] - Filtrer les composants (topologie), les événements et les traces dans votre vue

=== Perspectives

Les *Perspectives* d'une vue sont affichées sous forme d'onglets dans le coin supérieur gauche de l'interface utilisateur de la vue et vous permettent de visualiser l'ensemble des données d'une vue sous différents angles :

* xref:/use/views/k8s-overview-perspective.adoc[Perspective d'ensemble] ou xref:/use/views/k8s-highlights-perspective.adoc[Perspective des points forts] - selon le type de vue dans lequel vous vous trouvez
* xref:/use/views/k8s-topology-perspective.adoc[Perspective de topologie] - la carte des dépendances des composants de la vue
* xref:/use/views/k8s-events-perspective.adoc[Perspective des événements] - tous les événements se produisant sur la topologie
* xref:/use/views/k8s-metrics-perspective.adoc[Perspective des métriques] - indicateurs clés pour les composants les plus pertinents
* xref:/use/views/k8s-traces-perspective.adoc[Perspective des traces] - les informations de traçage s'exécutant sur la topologie

[NOTE]
====
*Toutes les perspectives* mettront à jour leur contenu en fonction de la configuration de la xref:/use/stackstate-ui/k8sTs-timeline-time-travel.adoc[timeline].
====
