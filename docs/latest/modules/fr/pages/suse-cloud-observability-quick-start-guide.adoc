== Présentation

Après avoir acheté SUSE Cloud Observability sur le Cloud Provider Marketplace, votre environnement SUSE Cloud Observability est provisionné.

Vous recevrez un e-mail de SUSE Cloud Observability avec les détails de connexion requis et des liens vers votre environnement. Ce guide de démarrage rapide vous aidera à commencer et à intégrer vos propres données dans votre déploiement SUSE Cloud Observability.

== Démarrer : Premiers pas

. Configurer un mot de passe
. Connectez-vous à votre instance SUSE Cloud Observability
. Installez l'agent SUSE Observability sur votre cluster
. Modifiez les détails de votre compte personnel

=== Configurer un mot de passe

L'e-mail de SUSE Cloud Observability contient un lien unique qui vous permet de définir votre mot de passe initial sur le compte.  Cela doit être effectué avant que vous puissiez vous connecter et configurer l'environnement d'observabilité.

=== Accéder à votre environnement SUSE Observability

Connectez-vous à votre environnement SUSE Cloud Observability en cliquant sur le lien unique dans l'e-mail que vous avez reçu de SUSE Cloud Observability.  Entrer votre mot de passe vous amènera à l'écran de configuration afin d'ajouter des clusters.

ifdef::feature-agent-install-cluster[]
=== Installez l'agent SUSE Observability sur votre cluster

SUSE Cloud Observability utilise des StackPacks pour faciliter la configuration de vos clusters en aval et l'intégration des données dans l'environnement d'observabilité.

Au départ, vous devriez être dirigé vers l'écran suivant, pour y accéder, dans l'interface utilisateur de SUSE Observability, ouvrez le menu principal en cliquant en haut à gauche de l'écran et allez à `StackPacks` > `Kubernetes`.

image::integrating_first_cluster_stackpacks.png[SUSE Cloud Observability - StackPacks]

Une fois que vous êtes dans l'écran StackPack Kubernetes, il est très simple d'ajouter un cluster à l'environnement d'observabilité.

Entrez un nom pour le cluster que vous souhaitez observer, il n'est pas nécessaire qu'il corresponde au nom du cluster utilisé dans 'kubeconfig'.

Le nom du cluster Kubernetes doit commencer et se terminer par une lettre minuscule ou un chiffre et ne peut être composé que de lettres minuscules, de chiffres, de points et de tirets (. -)

Entrez le nom et cliquez sur le bouton `Install`.  Cela devrait vous amener à l'écran suivant avec le cluster marqué comme 'En attente de données'

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - Ajout d’un Cluster]

Cliquez sur le nom du cluster ; cela développera cette section et révélera une série d'informations, y compris les prérequis et les commandes qui peuvent être utilisées sur vos clusters existants pour les ajouter à votre environnement d'observabilité.

Examinez les informations et assurez-vous d'avoir les autorisations correctes pour votre environnement Kubernetes ; assurez-vous que vous exécutez une version prise en charge de Kubernetes.

Ces commandes sont uniques à votre déploiement d'observabilité et incluent les clés API et les URL requises.  Sélectionnez les commandes appropriées pour votre cluster ; il existe des sections pour EKS, RKE, un cluster Kubernetes générique et bien d'autres.

Ces commandes installeront les agents SUSE Cloud Observability et connecteront le cluster à votre environnement SUSE Cloud Observability.

Vous pouvez utiliser les commandes directement depuis l'interface utilisateur ou déployer sur vos clusters en aval en suivant les commandes fournies dans la section 'Déployer l'Agent StackState et l'Agent de Cluster' dans le xref:/k8s-quick-start-guide.adoc[guide de démarrage rapide].

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - Post-installation]

Après que le cluster a été connecté, il devrait y avoir une coche verte dans l'interface utilisateur de SUSE Cloud Observability.

À ce stade, vous pouvez commencer à explorer vos données.
endif::feature-agent-install-cluster[]

=== Modifiez les détails de votre compte personnel

L'étape 4 de votre e-mail consiste à mettre à jour vos informations personnelles.  Cliquez sur le lien unique de l'e-mail et ajoutez vos informations personnelles de base si nécessaire.  Vous pouvez également configurer l'authentification 2FA depuis cette section si nécessaire.

=== Explorez vos données

Pour commencer à explorer vos données, ouvrez le menu principal en cliquant en haut à gauche de l'écran et allez à `Kubernetes` pour révéler une liste d'éléments observables.

image::accessing_views_1.png[SUSE Cloud Observability - Explorer vos données]

Sélectionnez 'Clusters' dans la section infrastructure qui devrait afficher une liste de clusters surveillés ; sélectionnez votre cluster pour révéler l'une des nombreuses vues intégrées.

image::accessing_views_2.png[SUSE Cloud Observability - Vue du Cluster]

À ce stade, vous pouvez commencer à explorer les données de votre cluster ou ajouter d'autres clusters à partir desquels collecter des données.

Pour plus d'informations sur l'utilisation de SUSE Cloud Observability, y compris la création de vues personnalisées, veuillez consulter la documentation standard.

=== Limitations de SUSE Cloud Observability

Notez que SUSE Cloud Observability ne fournit pas de signaux RED (Taux, Erreurs et Durée) prêts à l'emploi, seul le signal de Taux est affiché.  Cette fonctionnalité est disponible avec SUSE Rancher Prime.
Les clients qui ont besoin de ces signaux pour disposer d'une solution d'observabilité complète doivent contacter SUSE pour discuter de SUSE Rancher Prime, sinon ces données peuvent être collectées à l'aide des collecteurs OpenTelemetry.

== Informations complémentaires

Pour plus d'informations, y compris les prérequis et les versions Kubernetes prises en charge pour votre plateforme, veuillez vous référer au xref:/k8s-quick-start-guide.adoc[guide de démarrage rapide].
