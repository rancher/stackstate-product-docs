== 概要

一般的に、SUSE Observabilityにおけるxref:/use/views/k8s-view-structure.adoc[ビュー]の概念は、以前に保存したIT環境の特定の領域を監視できるようにします。しかし、特定のビューのサブセットを調査する必要がある場合、ビューの範囲を失いたくないことがよくあります。ここで*探索ビュー*が登場します。

特定のビュー（例：`my view`）の範囲を保持するために、トポロジ要素または要素の選択（例：コンポーネント、関係、グループ）に適用されるすべての調査アクションは、自動的に開始したビュー（例：`my view / explore`）の下にある一時的な探索ビューで開かれます。

自動的に探索ビューで開かれる*調査アクション*の例：

* *+ボタン*がxref:/use/views/k8s-topology-perspective.adoc[トポロジ視点]にあるコンポーネントでクリックされ、そのコンポーネントの隠れた隣接要素を表示しました。
* コンポーネントで*クイックアクション*が実行されました。
* トポロジ要素（コンポーネント、関係、グループ）が*ダブルクリックされ*、xref:/use/views/k8s-topology-perspective.adoc[トポロジ視点]で調査されました。
* トポロジ要素（コンポーネント、関係、グループ）で*'探索...'*リンクまたはボタンがクリックされ、調査されました。

[TIP]
====
既存の探索ビューから実行された調査アクションは、新しい探索ビューを*開かず*、現在の探索ビューの範囲を変更します。
====


== 取り扱い

探索ビューが自動的に作成されると、その元の範囲も定義されます。これは、いくつかの変更を加えた後に探索ビューを元の範囲にリセットしたい場合に役立ちます。

探索ビューは一時的なものであることを意図していますが、後で監視したりレビューしたりするためにxref:/use/views/k8s-custom-views.adoc[カスタムビュー]として保存することもできます。

探索ビューを保存したくない場合は、単にそれから離れるか、上部のナビゲーションバーのパンくずリストを使用して、開始したビューに戻ってください。
