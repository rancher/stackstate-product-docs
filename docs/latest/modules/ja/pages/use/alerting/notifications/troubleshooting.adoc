= 通知のトラブルシューティング
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

通知チャネルが通知を送信できない場合、通知の概要にエラーステータスが表示されます。エラーを確認するには：

. 通知をクリックします。
. ページの上部には、エラーの概要を示すバナーが表示されます。
. チャネルまでスクロールし、影響を受けたチャネルの「エラー」リンクをクリックしてエラーの詳細を開きます。

== よくあるエラー

=== Slack

Slackのエラーには通常、エラーコードが含まれます：

* `token_revoked`, `token_expired` または `missing_scope`：これらのエラーコードはすべて、SUSE ObservabilityがSlackにメッセージを投稿するための認証に問題があることを示しています。これを解決するには、チャネルを再作成してSUSE ObservabilityをSlackに再認証させ、必要なすべての権限が付与されるようにします。
* 他のエラーコードについては、https://api.slack.com/automation/cli/errors[Slack APIエラーコード]を参照してください。

=== HTTP

ほとんどのチャネル、特にWebhookは、ステータス200 OKのHTTPレスポンスを期待します。他のレスポンスはエラーと見なされます。HTTPエラーには、通常は短いメッセージと200 OKの代わりに受信したステータスコードが含まれます。

構成されたURLとトークンが正しいことを確認し、Webhookの場合はPOSTリクエストが適切に処理され、200 OKレスポンスを返すことを確認してください。

=== その他

エラーが「その他」のタイプの場合、SUSE Observabilityは外部サービスまたはWebhookに接続できない可能性が高いです。URLが正しいこと、解決できること、SUSE Observabilityによってアクセスできることを確認してください。
