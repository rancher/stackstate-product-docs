== Opsgenieの通知を設定する

Opsgenieに通知を送信するには、以下の手順に従ってください：

. <<_create_integration,OpsGenieで統合を作成する>>
. <<_create_and_test_the_channel,チャネルを作成する>>

=== 統合を作成する

SUSE ObservabilityをOpsGenieと統合するには、グローバルな「APIキー」を作成する必要があります。 通知の応答者は、各通知ごとに異なる場合があり、SUSE Observabilityで設定されます。

Opsgenieで、「設定 \-> 統合」に移動します。

* 「統合を追加」ボタンをクリックします。
* 「API」統合タイプを検索します。
* この統合の名前を選択しますが、_担当チームを選択しないでください_。

統合が作成されると、「APIキー」が生成されます。

=== チャネルを作成してテストする

image::k8s/notifications-opsgenie-channel-configuration.png[チャネルを作成する]

OpsGenieアカウントに適切な地域を選択し、「APIキー」を入力します。 通知の応答者を選択します；_ユーザー_、_チーム_、_スケジュール_、および_エスカレーション_が利用可能です。

「チャネルを追加」ボタンをクリックします。これにより、右側のチャネルリストにチャネルが追加されます。「テスト」ボタンが表示されます。それを押すと、OpsGenieでテストアラートが生成されます。

== 通知のためのOpsGenieアラート

通知が開かれると、新しいOpsGenieアラートが作成されます。このメッセージは、通常、健康状態が変化したときにのみ更新されます。通知が閉じられると、OpsGenieアラートも閉じられます。 アラートに関するメモには、閉じる理由が含まれています。
