== ログに移動する

SUSE Observability UIを通じてログ送信が設定された任意のKubernetes環境のコンテナログを探索できます。
これは、左側のKubernetesメニューから提供された任意のパラダイム（サービス、デプロイメント、ステートフルセット、デーモンセットなど）を通じて`pod`レベルに掘り下げることで行うことができます。

image::k8s/k8s-menu.png[Kubernetesパラダイムメニュー]

簡潔さと完全性のために、例では`Pods`パラダイムを直接使用し、複数のコンテナを含むポッドが選択されました。これにより、ポッドログとコンテナログを区別できるようになります。 ポッドビューには、複数のログ関連のエントリポイントがあります。

== ログを表示する

image::k8s/k8s-pod-view-node-agent.png[Kubernetesポッドビュー]

このポッドに関連するすべてのログを表示するには、赤で囲まれた「すべて表示」をクリックしてください。 ログエントリタイプ（エラー、警告、その他）でフィルタリングするには、青で囲まれた項目のいずれかを選択してください。 ログエントリタイプ間の比率を表示するヒストグラムは、緑で囲まれています。

=== ログドロワー

ログドロワーは、トラブルシューティング体験を向上させる方法でログファイルをナビゲートする、視覚的に優れた体験を提供します。

簡潔さのために、例ではフィルタリングされていない選択のみを扱い、すべてのログ行、すべてのコンテナ、任意のログエントリタイプを表示します。 `View all`を選択すると、フィルタリングが適用されていないログドロワーが開きます：

image::k8s/k8s-pod-view-log-drawer-no-filter.png[Kubernetesログビュー（フィルターなし）]

この時点から、提供されたフィルターにオプションを適用することで問題にさらに近づくことができます（検索文字列の一致、重大度、コンテナ、ログウィンドウの間隔）。 ログドロワーのヒストグラムのすぐ下には、ソートと表示の好みに関するいくつかのオプションがあります。

* 検索文字列の一致は、入力された検索用語を取り込み、それを含まないログ行をフィルタリングします。
* 重大度のオプションは`Any`、`Error`、`Warning`、および`Other`です。`Any`以外のものを選択すると、選択に一致しないすべてのログ行が除外されます。
* コンテナのオプションには`Any`とPod内のコンテナのリストが含まれます。 `Any`以外のものを選択すると、選択したコンテナによって生成された出力に制限されます。
* ログウィンドウの間隔オプションは、`from`と`to`の間隔を指定できる日付時刻ピッカーインターフェースになります。 デフォルトから変更すると、選択した期間外のすべてのログエントリが除外されます。

例として、以下ですべての4つが動作している様子をご覧いただけます。

image::k8s/k8s-pod-view-log-drawer-with-filters.png[Kubernetesログビュー（フィルター付き）]

上の画像には、以下のフィルターが適用されています。

* 検索語: `wrong type in json response`
* 重大度: `Warning`
* コンテナ: `node-agent`
* ログウィンドウの間隔:当日内の`02:00am`から`03:00am`までの間。
