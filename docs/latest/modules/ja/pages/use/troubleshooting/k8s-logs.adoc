== 概要

Kubernetesは、クラスターの活動とアプリケーションの出力に関する包括的なログを保持します。ログは、コンテナ、ノード、Kubernetes自体のパフォーマンスに関する貴重な洞察を提供します。PodsからReplicaSetsまで、すべてを綿密にログに記録することで、問題の原因を追跡することができます。

== ポッドログへのアクセス

ログ内のエラーや警告は、アプリケーション、コンテナ、またはポッド自体の潜在的な問題の指標です。ポッド画面でログを探るには、*ログを表示*ボタンをクリックしてください。

== ポッドログの使用

特定のエラーや警告をログ内で検索することができます。ログはコンテナによってフィルタリングできます。表示するログの希望の時間間隔と、ログをどのくらいの頻度で更新するかのリフレッシュ間隔を選択できます。

ログのヒストグラムは、選択した時間間隔内のログ行の数を視覚化し、クリックしてドラッグすることで特定の間隔にズームできます。

これらのオプションを使用して、各ログ行の追加情報を表示または非表示にします：

* *時間を表示*してタイムスタンプを表示
* *コンテナを表示*してログ行が属するコンテナ名を表示
* *行を折り返す*して長い行の折り返しを有効にする
* *新しい順/古い順*してログのソート順を切り替えます
