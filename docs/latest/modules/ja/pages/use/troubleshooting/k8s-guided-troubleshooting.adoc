== 概要

SUSE Observabilityによるガイド付きトラブルシューティングは、トラブルシューティングプロセス全体を通じてターゲットを絞った実行可能な洞察を提供することで、問題解決を加速する強力なアプローチです。高度なアルゴリズムを活用することで、SUSE Observabilityは、特定の環境に合わせたトラブルシューティングのヒント、視覚的支援、段階的なガイダンスを提供します。これにより、問題の特定と解決のプロセスが効率化されるだけでなく、サイト信頼性エンジニア（SRE）が開発チームをより良くサポートできるようになります。

SUSE Observabilityの修正ガイドを利用することで、エンジニアは一貫した高品質のサービスを確保し、他のチームメンバーと専門知識を共有できます。さらに、私たちの修正ガイドは、独自の環境に適応するために簡単に拡張または修正できるため、サービスの信頼性とパフォーマンスを維持するための貴重なツールとなります。

適切なものを監視し、適切なタイミングでアラートを発行する事前設定されたモニターは、エンジニアが問題を修正できるように明確なヒントで強化されています。このガイダンスは、すべてのエンジニアが修正のために何をすべきかを即座に理解するのに役立ちます。さらに、問題が解決された後、この情報は、改善が必要な点を特定するための非難のない事後分析のプロセスをサポートします。

== ガイド付きトラブルシューティングによる問題の修正

迅速に修正するために、SUSE Observabilityは、単一の画面にパッケージ化された明確な問題報告と修正ガイドを提供します。次の項目が含まれています：

. 問題をあまり知らない人に説明するための簡潔な説明。
. この問題に関する事実、例えば健康状態、トリガーされた時間、存在する場合の理由。
. サポートメトリックは、問題が時間とともにどのように進展したかを示します。
. しばしば、問題は孤立して発生することはありません。時には他の問題を引き起こしたり、実際の問題が別のコンポーネントによって引き起こされることもあります。SUSE Observabilityは、すべてのコンポーネントがどのように関連しているかを追跡し、関連する問題について警告します。
. 問題解決の各ステップを案内するための修正ガイドそのものです。

== トラブルシューティング時のピン留めされたアイテムの使用

トラブルシューティング中に修正ガイドを手元に置くために、それを*ピン留めされたアイテム*に追加できます。「*ピン留めアイテムに追加*」ボタンをクリックして、現在のモニターの修正ガイドをピン留めしてください。これで、トリガーされたモニターを閉じても、ステップバイステップのガイダンスに従うことができます。ピン留めされたアイテムメニューから、すべてのピン留めされた修正ガイドにアクセスできます。トラブルシューティングが完了したら、メニューからガイドのピン留めを外してください。
