= メトリクスを探索
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

SUSE Observabilityのメインメニューの下部に「メトリクスエクスプローラー」があります。これを使用して、任意のPromQLクエリを実行し、結果の時系列を視覚化します。クエリ結果は、選択した時間範囲のチャートと、時系列のラベルと共に最後の値を示すテーブルに表示されます。

image::k8s/k8s-metrics-explorer.png[メトリクスエクスプローラー]

== PromQLクエリ

ifdef::ss-ff-stackpacks2_enabled[]
クエリ入力フィールドには、メトリクス名、ラベル名と値、およびサポートされているPromQL関数の自動提案があります。完全なhttps://prometheus.io/docs/prometheus/latest/querying/basics/[PromQLガイドとリファレンス]については、Prometheusのドキュメントを参照してください。SUSE Observabilityは、任意のクエリで使用できる2つのデフォルトパラメータ：`${__interval}`と`${__rate_interval}`を追加します。これらは、チャートの解像度に応じて集約間隔を自動的にスケーリングするために使用できます（xref:/setup/custom-integrations/metric-bindings/writing-promql.adoc[詳細はこちら]）。
endif::[]

ifndef::ss-ff-stackpacks2_enabled[]
クエリ入力フィールドには、メトリクス名、ラベル名と値、およびサポートされているPromQL関数の自動提案があります。完全なhttps://prometheus.io/docs/prometheus/latest/querying/basics/[PromQLガイドとリファレンス]については、Prometheusのドキュメントを参照してください。SUSE Observabilityは、任意のクエリで使用できる2つのデフォルトパラメータ：`${__interval}`と`${__rate_interval}`を追加します。これらは、チャートの解像度に応じて集約間隔を自動的にスケーリングするために使用できます（xref:/use/metrics/k8s-writing-promql-for-charts.adoc[詳細はこちら]）。
endif::[]
