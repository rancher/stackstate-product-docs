= タイムラインと時間旅行
:revdate: 2025-07-10
:page-revdate: {revdate}
:description: SUSE Observability

== 概要

SUSE Observability UIの下部にあるタイムラインでは、過去の特定の時点におけるトポロジーの状態に時間を遡ることができます。その後、選択したトポロジースナップショットに対して利用可能なすべてのテレメトリをナビゲートできます。タイムラインの健康状態とイベントチャートは、選択したテレメトリ間隔中のトポロジーの状態の概要を示します。

image::k8s/k8s-timeline.png[タイムライン]

== タイムライン

=== テレメトリ間隔

テレメトリ間隔は、SUSE Observabilityの視点でイベント、メトリクス、トレースが利用可能な時間ウィンドウを指定します。それはタイムライン上で左から右に進みます。

image::v51_telemetry_interval.png[テレメトリ間隔]

選択したテレメトリ間隔は、相対的（ライブモード）であるか、カスタムテレメトリ間隔（タイムトラベルモード）に設定できます。デフォルトでは、テレメトリ間隔は相対的なテレメトリ間隔に設定されており、ライブモードで過去1時間のテレメトリを表示します。特定の時点のテレメトリを表示するために、ズームイン/アウトするか、カスタムテレメトリ間隔を設定できます。

==== テレメトリ間隔を設定する

[NOTE]
====

* テレメトリ間隔は最大6ヶ月に設定できます。
* テレメトリ間隔にカスタムテレメトリ間隔が設定されると、SUSE Observabilityは<<_topology_time,トポロジー時間>>を一時停止し、<<_time_travel,タイムトラベルモード>>に入ります。
====


テレメトリ間隔は以下の方法で設定できます：

* *ズームイン*

image::v51_timeline_click_drag.png[タイムライン上でクリックしてドラッグして、選択にカスタムテレメトリ間隔を設定します]

* *ズームアウト*

image::v51_telemetry_interval_zoom_out.png[虫眼鏡をクリックしてテレメトリ間隔のサイズを2倍にします]

* *テレメトリ間隔のジャンパー矢印を使用します*

image::v51_telemetry_interval_jumper.png[時間ジャンパー矢印をクリックしてテレメトリ間隔を時間の前後に移動させます]

* *相対的またはカスタムのテレメトリ間隔を設定します*

image::v51_timeline_telemetry_interval.png[ポップアップ「テレメトリ間隔を設定」を使用してテレメトリ間隔を指定します]

=== トポロジー時間

SUSE Observabilityのトポロジーは、その瞬間に観察された環境のスナップショットに基づいています。このスナップショットが取得される瞬間は、トポロジー時間によって指定されます。デフォルトでは、SUSE Observabilityはライブモードで、トポロジー時間は現在の時間に設定されています。カスタムのトポロジー時間を選択することで、トポロジーの以前の状態に<<_time_travel,タイムトラベル>>できます。これにより、特定の瞬間に問題を調査することができます。ポッドがすでに存在しなくても、どのように接続されていたか、そのログ、イベント、関連リソース、イベントなどを見ることができます。

image::v51_topology_time.png[トポロジー時間]

タイムライン上で、選択されたトポロジー時間はプレイヘッドによって示されます - 現在のトポロジー時間が上部にある黒い線です。また、タイムラインの上部にある*トポロジー時間*ボックスにも指定されています。

==== トポロジー時間を設定します

トポロジー時間は以下の方法で設定できます：

* *タイムラインをクリックします*

image::v51_topology_time_timeline.png[タイムラインの任意の場所をクリックして、その瞬間にトポロジー時間を設定します]

* *トポロジー時間のジャンパー矢印を使用します*

image::v51_topology_time_jumper.png[トポロジーの時間ジャンパー矢印をクリックして、トポロジーの時間を次のイベントセットに向けて前後に移動します]

* *カスタムトポロジー時間を設定する*

image::v51_topology_time_popup.png[ポップアップ「トポロジー時間を設定」を使用して、トポロジー時間を指定します]

* *タイムスタンプをクリックしてください*

image::v51_topology_time_timestamp.png[タイムスタンプをクリックして、その特定のトポロジー時間にジャンプしてください]

==== テレメトリ間隔外のトポロジー時間

選択したトポロジー時間が現在選択されている<<_telemetry_interval,テレメトリ間隔>>の外の時間である場合、「トポロジー時間は現在のテレメトリ間隔外です」というメッセージが表示され、タイムラインの上部にある*トポロジー時間*ボックスが黒くハイライトされます。タイムラインは左から右にテレメトリ間隔を示しているため、現在のトポロジー時間を示すプレイヘッドはタイムライン上に表示されません。

image::v51_topology_time_outside_telemetry_interval.png[テレメトリ間隔外のトポロジー時間]

トポロジーとテレメトリを期待通りにブラウズすることができます：

* xref:/use/views/k8s-topology-perspective.adoc[トポロジーの視点]では、選択したトポロジー時間におけるトポロジーの状態が視覚化されています。
* すべての視点において、選択したテレメトリ間隔で生成されたテレメトリが表示され、選択したトポロジー時間に存在したトポロジー要素に関連しています。

==== ライブモード

時間旅行を停止し、トポロジー時間をライブモードに戻すには、画面の上部にある*ライブに戻る*または*ライブに戻る*をクリックしてください。

image::v51_timeline_go_live.png[ライブに戻る]

=== ヘルス

選択したテレメトリ間隔中のビューの健康状態は、タイムラインの*健康*ラインに色として表示されます。

単一のリソースの場合、健康状態は時間の経過とともに表示され、概要では灰色の線が表示されます。

image::v51_timeline_no_health_state.png[健康状態は利用できません]

=== ［Events (イベント)］

タイムラインの*イベント*ラインは、各時点で生成されたイベントの数を示す棒グラフを表示します。これにより、過去の活動が多かった瞬間を確認することができます。選択した<<_topology_time,トポロジー時間>>に存在していたトポロジー要素によって生成されたイベントのみが表示されます。

興味のあるイベントバーをズームインするには、タイムライン上でそれを囲む小さなテレメトリ間隔を選択するためにクリックしてドラッグします。

image::v51_timeline_click_drag.png[クリックしてドラッグしてテレメトリ間隔を選択します]

[NOTE]
====
タイムライン上での単一のクリックは、再生ヘッドをこの時点に移動させ、選択した<<_topology_time,トポロジー時間>>におけるトポロジーの状態にタイムトラベルします。新たに選択されたトポロジー時間に存在していたトポロジー要素によって生成されたイベントのみが表示されます。
====


== タイムトラベル

各SUSE Observabilityの視点では、ライブモードまたは過去のモードのいずれかにいることができます。ライブモードでは、SUSE Observabilityは新しいデータを常にポーリングします。トポロジーやテレメトリを通じてタイムトラベルすると、実際にはインフラストラクチャのスナップショットを扱っていることになります。利用可能なデータは、2つの選択に基づいています：

* <<_topology_time,トポロジー時間>> - Kubernetesリソースのスナップショットを取得したい特定の時点。
* <<_telemetry_interval,テレメトリ間隔>> - テレメトリとトレースを表示したい時間範囲。

タイムトラベルを停止してライブモードに戻るには、画面の上部にある*ライブに戻る*または*ライブに戻る*をクリックします。

image::v51_timeline_go_live.png[ライブに戻る]
