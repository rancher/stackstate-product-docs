== 概要

クラウドプロバイダーのマーケットプレイスから SUSE Cloud Observability を購入した後、あなたの SUSE Cloud Observability 環境がプロビジョニングされます。

SUSE Cloud Observability から必要なログイン詳細と環境へのリンクが記載されたメールが届きます。このクイックスタートガイドは、あなたが始める手助けをし、あなた自身のデータを SUSE Cloud Observability のデプロイメントに取り込むためのものです。

== はじめに - 最初のステップ

. パスワードを設定する
. SUSE Cloud Observability インスタンスにログインする
. クラスターに SUSE Observability エージェントをインストールする
. 個人アカウントの詳細を変更する

=== パスワードを設定する

SUSE Cloud Observability からのメールには、アカウントの初期パスワードを設定するためのユニークなリンクが含まれています。 これは、ログインして監視環境を構成する前に実行する必要があります。

=== SUSE Observability 環境にアクセスする

SUSE Cloud Observability から受け取ったメールのユニークなリンクをクリックして、SUSE Cloud Observability 環境にログインします。 パスワードを入力すると、クラスターを追加するための設定画面に移動します。

ifdef::feature-agent-install-cluster[]
=== クラスターに SUSE Observability エージェントをインストールする

SUSE Cloud Observability は、下流のクラスターを構成し、データを監視環境に取り込むのを容易にするために StackPacks を使用します。

最初に、次の画面に移動するはずです。SUSE Observability UI で、画面の左上をクリックしてメインメニューを開き、`StackPacks` > `Kubernetes` に進みます。

image::integrating_first_cluster_stackpacks.png[SUSE Cloud Observability - StackPacks]

Kubernetes StackPack の画面に入ると、監視環境にクラスターを追加するのは非常に簡単です。

監視したいクラスターの名前を入力してください。'kubeconfig' で使用されるクラスター名と一致する必要はありません。

Kubernetes クラスター名は小文字の文字または数字で始まり、終わる必要があり、小文字の文字、数字、ドット、ダッシュ（. -）のみで構成できます。

名前を入力し、`Install`ボタンをクリックします。 これにより、クラスターが「データ待機中」とフラグ付けされた次の画面に移動します。

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - クラスターの追加]

クラスター名をクリックすると、このセクションが展開され、前提条件や、既存のクラスターに追加するために使用できるコマンドの一連の情報が表示されます。

情報を確認し、Kubernetes環境への正しい権限を持っていることを確認し、サポートされているKubernetesのバージョンを実行していることを確認してください。

これらのコマンドは、あなたの監視デプロイメントに固有で、必要な API キーと URL が含まれています。 クラスターに適切なコマンドを選択してください。EKS、RKE、一般的なKubernetesクラスターなどのセクションがあります。

これらのコマンドは、SUSE Cloud Observabilityエージェントをインストールし、クラスターをSUSE Cloud Observability環境に接続します。

UIからコマンドを直接使用することもできますし、xref:/k8s-quick-start-guide.adoc[クイックスタートガイド]の「StackStateエージェントとクラスターエージェントをデプロイする」セクションに記載されているコマンドに従って、ダウンストリームのクラスターにデプロイすることもできます。

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - インストール後]

クラスターが接続された後、SUSE Cloud Observability UIに緑のチェックマークが表示されるはずです。

この時点で、データの探索を開始できます。
endif::feature-agent-install-cluster[]

=== 個人アカウントの詳細を変更する

メールのステップ4は、個人情報を更新することです。 メールのユニークなリンクをクリックし、必要に応じて基本的な個人情報を追加してください。 必要に応じて、このセクションから2FA認証を設定することもできます。

=== データを探索する

データを探索するには、画面の左上をクリックしてメインメニューを開き、`Kubernetes`に移動して監視可能なアイテムのリストを表示します。

image::accessing_views_1.png[SUSE Cloud Observability - データの探索]

インフラストラクチャセクションから「クラスター」を選択すると、監視されているクラスターのリストが表示されます。クラスターを選択して、多くのビルトインビューの1つを表示します。

image::accessing_views_2.png[SUSE Cloud Observability - クラスタービュー]

この時点で、クラスターのデータを探索したり、データを収集するための他のクラスターを追加したりすることができます。

SUSE Cloud Observabilityの使用方法についての詳細情報、カスタムビューの作成を含む、標準のドキュメントをご覧ください。

=== SUSE Cloud Observabilityの制限事項

SUSE Cloud Observabilityは、標準でRED信号（レート、エラー、期間）を提供していないことに注意してください。表示されるのはレート信号のみです。 この機能はSUSE Rancher Primeで利用可能です。
完全な監視ソリューションにこれらの信号が必要な顧客は、SUSEに連絡してSUSE Rancher Primeについて相談するか、OpenTelemetryコレクターを使用してこのデータを収集することができます。
