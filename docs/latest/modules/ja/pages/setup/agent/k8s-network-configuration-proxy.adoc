SUSE Observability Kubernetes エージェントは、発信する接続に対して HTTP または HTTPS のプロキシ設定を構成できるようになっています。

== SUSE Observability との通信用プロキシ

エージェントをSUSE Observability バックエンドへの接続をプロキシするように設定するには、Helm の設定を使用できます。

=== Helm 設定

==== `values.yaml` ファイルを介して

. Helm チャートの `values.yaml` ファイルを開いてください。
. `global.proxy.url` 設定を見つけて、プロキシ URL を指定します：
+
[,yaml]
----
 global:
   proxy:
     url: "https://proxy.example.com:8080"
----

. オプションで、プロキシに署名された証明書がない場合は、`global.skipSslValidation` を `true` に設定して SSL 検証を無効にします：
+
[,yaml]
----
 global:
   skipSslValidation: true
----

==== コマンドラインフラグを介して

. Helm チャートのインストール中に、`--set` フラグを使用してプロキシ URL を指定します：
+
[,bash]
----
 helm install suse-observability-agent suse-observability/suse-observability-agent --set global.proxy.url="https://proxy.example.com:8080"
----

. コマンドラインを介して SSL 検証を無効にするには、次を使用します：
+
[,bash]
----
 helm install suse-observability-agent suse-observability/suse-observability-agent --set global.skipSslValidation=true
----
