== 概要

ここに文書化されたコンポーネントプレゼンテーションモデルは、コンポーネント（例：ポッド、VM、サービス）が{stackstate-product-name}ユーザーインターフェースにどのように表示されるかを宣言します。
それは、命名、アイコン、テーブル、概要およびハイライトページ、フィルター、そしてメニューの配置などの*プレゼンテーションに関する側面*に専念しています。

このモデルの目標は、既存のより断片的なアプローチと比較して、コンポーネントのUI動作を定義するためのより構成可能で拡張可能な方法を提供することです。
