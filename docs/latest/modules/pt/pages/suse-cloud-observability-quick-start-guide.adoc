== Visão Geral

Após adquirir o SUSE Cloud Observability no Marketplace do Provedor de Nuvem, seu ambiente do SUSE Cloud Observability é provisionado.

Você receberá um e-mail do SUSE Cloud Observability com os detalhes de login necessários e links para seu ambiente. Este guia de início rápido ajudará você a começar e a inserir seus próprios dados na sua implantação do SUSE Cloud Observability.

== Começando - Primeiros Passos

. Configurar uma senha
. Faça login na sua instância do SUSE Cloud Observability
. Instale o agente do SUSE Observability no seu cluster
. Altere os detalhes da sua conta pessoal

=== Configurar uma senha

O e-mail do SUSE Cloud Observability contém um link único que permite definir sua senha inicial na conta.  Isso deve ser realizado antes que você possa fazer login e configurar o ambiente de observabilidade.

=== Acessando seu Ambiente do SUSE Observability

Faça login no seu ambiente do SUSE Cloud Observability clicando no link único no e-mail que você recebeu do SUSE Cloud Observability.  Inserir sua senha levará você à tela de configuração para adicionar clusters.

ifdef::feature-agent-install-cluster[]
=== Instale o agente do SUSE Observability no seu cluster

O SUSE Cloud Observability utiliza StackPacks para facilitar a configuração dos seus clusters downstream e a inserção de dados no ambiente de observabilidade.

Inicialmente, você deve ser levado à seguinte tela; para chegar aqui, na interface do SUSE Observability, abra o menu principal clicando no canto superior esquerdo da tela e vá para `StackPacks` > `Kubernetes`.

image::integrating_first_cluster_stackpacks.png[SUSE Cloud Observability - StackPacks]

Uma vez na tela do StackPack do Kubernetes, é muito simples adicionar um cluster ao ambiente de observabilidade.

Digite um nome para o cluster que você deseja observar; ele não precisa corresponder ao nome do cluster usado no 'kubeconfig'.

O nome do cluster Kubernetes deve começar e terminar com uma letra minúscula ou dígito e pode consistir apenas de letras minúsculas, dígitos, pontos e traços (. -)

Digite o nome e clique no botão `Install`.  Isso deve levá-lo à seguinte tela com o cluster marcado como 'Aguardando dados'

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - Adicionando um Cluster]

Clique no nome do cluster, isso expandirá esta seção e revelará uma série de informações, incluindo pré-requisitos e comandos que podem ser usados em seus clusters existentes para adicioná-los ao seu ambiente de observabilidade.

Revise as informações e certifique-se de que você tem as permissões corretas para o seu ambiente Kubernetes, assegure-se de que está executando uma versão suportada do Kubernetes.

Esses comandos são exclusivos para sua implantação de observabilidade e incluem as chaves de API e URLs necessárias.  Selecione os comandos apropriados para o seu cluster, há seções para EKS, RKE, cluster Kubernetes genérico e muitos mais.

Esses comandos instalarão os agentes do SUSE Cloud Observability e conectarão o cluster ao seu ambiente do SUSE Cloud Observability.

Você pode usar os comandos diretamente da interface do usuário ou pode implantar em seus clusters downstream seguindo os comandos fornecidos na seção 'Implantar o Agente StackState e o Agente do Cluster' no xref:/k8s-quick-start-guide.adoc[guia de início rápido].

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - Pós-instalação]

Após a conexão do cluster, deve haver um sinal verde na interface do usuário do SUSE Cloud Observability.

Neste ponto, você pode começar a explorar seus dados.
endif::feature-agent-install-cluster[]

=== Altere os detalhes da sua conta pessoal

O passo 4 do seu e-mail é atualizar suas informações pessoais.  Clique no link exclusivo do e-mail e adicione seus dados pessoais básicos conforme necessário.  Você também pode configurar a autenticação 2FA nesta seção, se necessário.

=== Explore seus dados

Para começar a explorar seus dados, abra o menu principal clicando no canto superior esquerdo da tela e vá para `Kubernetes` para revelar uma lista de itens observáveis.

image::accessing_views_1.png[SUSE Cloud Observability - Explorando Seus Dados]

Selecione 'Clusters' na seção de infraestrutura, que deve mostrar uma lista de clusters monitorados, selecione seu cluster para revelar uma das muitas visualizações incorporadas.

image::accessing_views_2.png[SUSE Cloud Observability - Visualização do Cluster]

Neste ponto, você pode começar a explorar os dados do seu cluster ou adicionar mais clusters dos quais coletar dados.

Para mais informações sobre como usar o SUSE Cloud Observability, incluindo a criação de visualizações personalizadas, consulte a documentação padrão.

=== Limitações do SUSE Cloud Observability

Observe que o SUSE Cloud Observability não fornece sinais RED (Taxa, Erros e Duração) prontos para uso; apenas o sinal de Taxa é exibido.  Este recurso está disponível com o SUSE Rancher Prime.
Clientes que precisam desses sinais para ter uma solução completa de Observabilidade devem entrar em contato com a SUSE para discutir o SUSE Rancher Prime; alternativamente, esses dados podem ser coletados usando os coletores OpenTelemetry.

== Informações adicionais

Para mais informações, incluindo pré-requisitos e versões suportadas do Kubernetes para sua plataforma, consulte o xref:/k8s-quick-start-guide.adoc[guia de início rápido].
