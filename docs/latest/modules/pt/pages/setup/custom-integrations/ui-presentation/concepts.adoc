== Apresentação de Componente

O conceito central do modelo de Apresentação de UI é a *Apresentação de Componente*.

Uma `ComponentPresentation` define como um ou mais componentes correspondentes são apresentados na UI, dependendo do contexto ativo.
Ele se concentra exclusivamente em *questões de apresentação*, como nomes, ícones, tabelas, páginas de visão geral e destaque, filtros e posicionamento de itens de menu.

Em um nível alto, uma Apresentação de Componente:

* Seleciona componentes usando uma ligação (uma consulta STQL simples).
* Aplica dentro de um modo específico (por exemplo, observabilidade).
* Define um ou mais aspectos de apresentação (como tabelas de visão geral ou destaques).
* Pode definir parcialmente propriedades de apresentação e compor com outras definições correspondentes.

Várias Apresentações de Componente podem se aplicar ao mesmo componente. Nesses casos, suas definições são combinadas para determinar a apresentação final.
