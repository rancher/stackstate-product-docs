== Visão Geral

Uma view no SUSE Observability permite que você monitore e inspecione um subconjunto do seu ambiente de TI. A estrutura de uma view é adaptada para filtrar e visualizar os dados nesse subconjunto (view) de maneira eficiente.

=== Filtros

O menu *Filtros* no canto superior direito da interface da view permite que você filtre os componentes (topologia), eventos e rastreamentos exibidos em uma view. Uma vez aplicados, os filtros afetarão o conteúdo de todas as perspectivas em uma view.

* xref:/use/views/k8s-filters.adoc[Filtros] - Filtrar os componentes (topologia), eventos e rastreamentos na sua view

=== Perspectivas

As *Perspectivas* de uma view são exibidas como abas no canto superior esquerdo da interface da view e permitem que você visualize todos os dados em uma view através de diferentes lentes:

* xref:/use/views/k8s-overview-perspective.adoc[Perspectiva de visão geral] ou xref:/use/views/k8s-highlights-perspective.adoc[Perspectiva de destaques] - dependendo do tipo de view em que você está
* xref:/use/views/k8s-topology-perspective.adoc[Perspectiva de topologia] - o mapa de dependência dos componentes da view
* xref:/use/views/k8s-events-perspective.adoc[Perspectiva de eventos] - todos os eventos que ocorrem na topologia
* xref:/use/views/k8s-metrics-perspective.adoc[Perspectiva de métricas] - métricas chave para os componentes mais relevantes
* xref:/use/views/k8s-traces-perspective.adoc[Perspectiva de rastreamentos] - as informações de rastreamento que ocorrem na topologia

[NOTE]
====
*Todas as perspectivas* atualizarão seu conteúdo com base na configuração da xref:/use/stackstate-ui/k8sTs-timeline-time-travel.adoc[linha do tempo].
====
