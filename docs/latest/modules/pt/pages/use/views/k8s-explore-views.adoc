== Visão Geral

De modo geral, o conceito de uma xref:/use/views/k8s-view-structure.adoc[visualização] na SUSE Observability permite que você monitore uma área do seu ambiente de TI que você havia salvo anteriormente. Mas, muitas vezes, quando você precisa investigar um subconjunto de uma visualização específica, não quer perder o escopo dessa visualização. É aqui que as *visualizações de exploração* entram em cena.

Para manter o escopo de uma visualização específica (por exemplo, `my view`) intacto, todas as ações investigativas aplicadas a um elemento de topologia ou seleção de elementos (por exemplo, componentes, relações, grupos) serão automaticamente abertas em uma visualização de exploração temporária, sob a visualização de onde você partiu (por exemplo, `my view / explore`).

Exemplos de *ações investigativas* que serão automaticamente abertas em uma visualização de exploração:

* O *botão +* foi clicado em um componente na xref:/use/views/k8s-topology-perspective.adoc[perspectiva de topologia] para mostrar os vizinhos ocultos desse componente.
* Uma *ação rápida* foi executada em um componente.
* Um elemento de topologia (componente, relação, grupo) foi *clicado duas vezes* na xref:/use/views/k8s-topology-perspective.adoc[perspectiva de topologia] para investigá-lo.
* O link ou botão *'Explorar...'* foi clicado em um elemento de topologia (componente, relação, grupo) para investigá-lo.

[TIP]
====
As ações investigativas executadas a partir de uma visualização de exploração existente *não* abrirão uma nova visualização de exploração – elas mudarão o escopo da visualização de exploração em que você está.
====


== Gerenciamento

Quando uma visualização de exploração é criada automaticamente, seu escopo original também é definido. Isso é útil se você quiser redefinir a visualização de exploração para seu escopo original após ter feito algumas alterações nela.

Embora uma visualização de exploração seja destinada a ser temporária, ela pode ser salva como uma xref:/use/views/k8s-custom-views.adoc[visualização personalizada] se você quiser monitorá-la ou revisá-la mais tarde.

Se você não quiser salvar uma visualização de exploração, basta se afastar dela ou voltar para a visualização de onde você partiu usando as migalhas de pão na barra de navegação superior.

== Estrutura

As visualizações de exploração têm uma estrutura idêntica às xref:/use/views/k8s-custom-views.adoc[visualizações personalizadas]: elas têm os mesmos xref:/use/views/k8s-view-structure.adoc#_filters[filtros] e as mesmas xref:/use/views/k8s-view-structure.adoc#_perspectives[perspectivas].
