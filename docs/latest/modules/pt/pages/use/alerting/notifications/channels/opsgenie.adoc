== Configurar notificações do OpsGenie

Para enviar notificações para o Opsgenie, siga estas etapas:

. <<_create_integration,Criar uma integração no OpsGenie>>
. <<_create_and_test_the_channel,Criar o canal>>

=== Criar integração

Para integrar o SUSE Observability com o OpsGenie, uma "chave de API" global precisa ser criada.  Os respondentes de uma notificação podem ser diferentes para cada notificação e serão configurados no SUSE Observability.

No OpsGenie, vá para "Configurações \-> Integrações".

* Clique no botão "Adicionar integração"
* Pesquise pelo tipo de integração "API".
* Escolha um nome para esta integração, mas _não selecione uma equipe responsável_.

Quando a integração for criada, ela terá uma "chave de API".

=== Criar e testar o canal

image::k8s/notifications-opsgenie-channel-configuration.png[Criar o canal]

Selecione a região apropriada para sua conta do OpsGenie e insira a "chave de API".  Escolha os respondentes para a notificação; _usuários_, _equipes_, _escalas_ e _escalonamentos_ estão disponíveis para isso.

Clique no botão "Adicionar canal". Isso adiciona o canal à lista de canais à direita. Ele mostrará um botão "Testar". Pressionar isso gerará um alerta de teste no OpsGenie.

== Alertas do OpsGenie para notificações

Quando uma notificação é aberta, um novo alerta do OpsGenie é criado. Esta mensagem será atualizada para mudanças, geralmente apenas quando o estado de saúde muda. Quando a notificação é fechada, o alerta do OpsGenie é fechado novamente.  Uma nota no alerta contém o motivo para fechá-lo.
