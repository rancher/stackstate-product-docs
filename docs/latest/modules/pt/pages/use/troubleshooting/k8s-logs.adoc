== Visão Geral

O Kubernetes mantém logs abrangentes da atividade do cluster e da saída do aplicativo. O registro de logs oferece informações valiosas sobre como os contêineres, nós e o próprio Kubernetes estão se comportando. Registrando meticulosamente tudo, desde Pods até ReplicaSets, permite que você rastreie problemas até sua origem.

== Acessando logs de pods

Erros e avisos nos logs são indicadores de problemas potenciais em seu aplicativo, contêineres ou no próprio pod. Para explorar os logs na tela do pod, clique no botão *Mostrar logs*.

== Usando logs de pods

Você pode usar a busca para procurar erros ou avisos específicos nos logs. Os logs podem ser filtrados por um contêiner. Você pode selecionar um intervalo de tempo desejado para os logs que serão exibidos e um intervalo de atualização para a frequência com que deseja que os logs sejam atualizados.

O histograma de logs visualiza o número de linhas de log dentro de um intervalo de tempo selecionado e permite que você amplie para um intervalo específico clicando e arrastando.

Use estas opções para mostrar ou ocultar informações adicionais para cada linha de log:

* *Mostrar hora* para mostrar timestamps
* *Mostrar contêineres* para mostrar o nome do contêiner ao qual a linha de log pertence
* *Quebrar linhas* para habilitar a quebra de linhas longas
* *Mais recentes primeiro/Antigos primeiro* para alternar a ordem de classificação dos logs
