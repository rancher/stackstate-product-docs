== Übersicht

Im Allgemeinen ermöglicht das Konzept einer xref:/use/views/k8s-view-structure.adoc[Ansicht] in SUSE Observability, einen Bereich Ihrer IT-Landschaft zu überwachen, den Sie zuvor gespeichert haben. Aber oft, wenn Sie einen Teil einer bestimmten Ansicht untersuchen müssen, möchten Sie den Umfang dieser Ansicht nicht verlieren. Hier kommen die *Erkundungsansichten* ins Spiel.

Um den Umfang einer bestimmten Ansicht (z. B. `my view`) intakt zu halten, werden alle Untersuchungsaktionen, die auf ein Topologieelement oder eine Auswahl von Elementen (z. B. Komponenten, Beziehungen, Gruppen) angewendet werden, automatisch in einer temporären Erkundungsansicht geöffnet, unter der Ansicht, von der Sie gestartet sind (z. B. `my view / explore`).

Beispiele für *Untersuchungsaktionen*, die automatisch in einer Erkundungsansicht geöffnet werden:

* Der *+ Button* wurde auf einer Komponente in der xref:/use/views/k8s-topology-perspective.adoc[Topologie-Perspektive] angeklickt, um die versteckten Nachbarn dieser Komponente anzuzeigen.
* Eine *Schnellaktion* wurde auf einer Komponente ausgeführt.
* Ein Topologieelement (Komponente, Beziehung, Gruppe) wurde *doppelt geklickt* in der xref:/use/views/k8s-topology-perspective.adoc[Topologie-Perspektive], um es zu untersuchen.
* Der *'Erkunden...'* Link oder Button wurde auf einem Topologieelement (Komponente, Beziehung, Gruppe) angeklickt, um es zu untersuchen.

[TIP]
====
Die Untersuchungsaktionen, die aus einer bestehenden Erkundungsansicht ausgeführt werden, werden *nicht* eine neue Erkundungsansicht öffnen - sie ändern den Umfang der Erkundungsansicht, in der Sie sich befinden.
====


== Handhabung

Wenn eine Erkundungsansicht automatisch erstellt wird, wird auch ihr ursprünglicher Umfang definiert. Das ist hilfreich, wenn Sie die Erkundungsansicht auf ihren ursprünglichen Umfang zurücksetzen möchten, nachdem Sie einige Änderungen daran vorgenommen haben.

Obwohl eine Erkundungsansicht vorübergehend sein soll, kann sie als xref:/use/views/k8s-custom-views.adoc[benutzerdefinierte Ansicht] gespeichert werden, wenn Sie sie später überwachen oder überprüfen möchten.

Wenn Sie eine Erkundungsansicht nicht speichern möchten, bewegen Sie sich einfach davon weg oder kehren Sie zu der Ansicht zurück, von der Sie gestartet sind, indem Sie die Brotkrumen in der oberen Navigationsleiste verwenden.

== Struktur

Die Erkundungsansichten haben eine identische Struktur wie die xref:/use/views/k8s-custom-views.adoc[benutzerdefinierte Ansichten]: sie haben die gleichen xref:/use/views/k8s-view-structure.adoc#_filters[Filter] und die gleichen xref:/use/views/k8s-view-structure.adoc#_perspectives[Perspektiven].
