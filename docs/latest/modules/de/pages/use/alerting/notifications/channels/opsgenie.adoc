== Opsgenie-Benachrichtigungen konfigurieren

Um Benachrichtigungen an Opsgenie zu senden, befolgen Sie diese Schritte:

. <<_create_integration,Erstellen Sie eine Integration in OpsGenie>>
. <<_create_and_test_the_channel,Erstellen Sie den Kanal>>

=== Integration erstellen

Um SUSE Observability mit OpsGenie zu integrieren, muss ein globaler "API-Schlüssel" erstellt werden.  Die Verantwortlichen für eine Benachrichtigung können für jede Benachrichtigung unterschiedlich sein und werden in SUSE Observability konfiguriert.

Gehen Sie in Opsgenie zu "Einstellungen \-> Integrationen".

* Klicken Sie auf die Schaltfläche "Integration hinzufügen"
* Suchen Sie nach dem Integrations-Typ "API".
* Wählen Sie einen Namen für diese Integration, aber _wählen Sie kein zugewiesenes Team_.

Wenn die Integration erstellt wurde, hat sie einen "API-Schlüssel".

=== Erstellen und testen Sie den Kanal

image::k8s/notifications-opsgenie-channel-configuration.png[Erstellen Sie den Kanal]

Wählen Sie die entsprechende Region für Ihr OpsGenie-Konto aus und geben Sie den "API-Schlüssel" ein.  Wählen Sie Verantwortliche für die Benachrichtigung aus; _Benutzer_, _Teams_, _Zeitpläne_ und _Eskalationen_ sind dafür verfügbar.

Klicken Sie auf die Schaltfläche "Kanal hinzufügen". Dies fügt den Kanal zur Liste der Kanäle auf der rechten Seite hinzu. Es wird eine "Test"-Schaltfläche angezeigt. Wenn Sie darauf drücken, wird eine Testwarnung in OpsGenie generiert.

== OpsGenie-Warnungen für Benachrichtigungen

Wenn eine Benachrichtigung geöffnet wird, wird eine neue OpsGenie-Warnung erstellt. Diese Nachricht wird bei Änderungen aktualisiert, normalerweise nur wenn sich der Gesundheitszustand ändert. Wenn die Benachrichtigung geschlossen wird, wird die OpsGenie-Warnung erneut geschlossen.  Eine Notiz zur Warnung enthält den Grund für die Schließung.
