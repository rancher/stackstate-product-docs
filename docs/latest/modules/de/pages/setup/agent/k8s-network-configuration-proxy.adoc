= Konfiguration des SUSE Observability Kubernetes Agenten für Proxy-Verbindungen
:revdate: 2025-07-10
:page-revdate: {revdate}

Der SUSE Observability Kubernetes Agent ermöglicht es Ihnen, HTTP- oder HTTPS-Proxy-Einstellungen für die Verbindungen, die er initiiert, zu konfigurieren.

== Proxy für die Kommunikation mit SUSE Observability

Um den Agenten so zu konfigurieren, dass er Verbindungen zum SUSE Observability-Backend über einen Proxy herstellt, können Sie die Helm-Konfiguration verwenden.

=== Helm-Konfiguration

==== Über die `values.yaml` Datei

. Öffnen Sie Ihre Helm-Chart `values.yaml` Datei.
. Suchen Sie die `global.proxy.url` Konfiguration und geben Sie die Proxy-URL an:
+
[,yaml]
----
 global:
   proxy:
     url: "https://proxy.example.com:8080"
----

. Optional, wenn der Proxy kein signiertes Zertifikat hat, deaktivieren Sie die SSL-Überprüfung, indem Sie `global.skipSslValidation` auf `true` setzen:
+
[,yaml]
----
 global:
   skipSslValidation: true
----

==== Über das Befehlszeilen-Flag

. Verwenden Sie während der Installation des Helm-Charts den `--set` Flag, um die Proxy-URL anzugeben:
+
[,bash]
----
 helm install suse-observability-agent suse-observability/suse-observability-agent --set global.proxy.url="https://proxy.example.com:8080"
----

. Um die SSL-Überprüfung über die Befehlszeile zu deaktivieren, verwenden Sie:
+
[,bash]
----
 helm install suse-observability-agent suse-observability/suse-observability-agent --set global.skipSslValidation=true
----
