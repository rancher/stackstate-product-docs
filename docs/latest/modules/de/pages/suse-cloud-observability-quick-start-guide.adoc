:description: SUSE Cloud Observability

== Übersicht

Nach dem Kauf von SUSE Cloud Observability im Cloud Provider Marketplace wird Ihre SUSE Cloud Observability-Umgebung bereitgestellt.

Sie erhalten eine E-Mail von SUSE Cloud Observability mit den erforderlichen Anmeldedaten und Links zu Ihrer Umgebung. Diese Kurzanleitung hilft Ihnen, loszulegen und Ihre eigenen Daten in Ihre SUSE Cloud Observability-Implementierung zu integrieren.

== Erste Schritte

. Richten Sie ein Passwort ein.
. Melden Sie sich bei Ihrer SUSE Cloud Observability-Instanz an
. Installieren Sie den SUSE Observability-Agent in Ihrem Cluster
. Ändern Sie Ihre persönlichen Kontodaten

=== Richten Sie ein Passwort ein.

Die E-Mail von SUSE Cloud Observability enthält einen einzigartigen Link, der es Ihnen ermöglicht, Ihr anfängliches Passwort für das Konto festzulegen.  Dies muss durchgeführt werden, bevor Sie sich anmelden und die Observability-Umgebung konfigurieren können.

=== Zugriff auf Ihre SUSE Cloud Observability-Umgebung

Melden Sie sich in Ihrer SUSE Cloud Observability-Umgebung an, indem Sie auf den einzigartigen Link in der E-Mail klicken, die Sie von SUSE Cloud Observability erhalten haben.  Die Eingabe Ihres Passworts führt Sie zum Konfigurationsbildschirm, um Cluster hinzuzufügen.

ifdef::feature-agent-install-cluster[]
=== Installieren Sie den SUSE Observability-Agent in Ihrem Cluster

SUSE Cloud Observability verwendet StackPacks, um die Konfiguration Ihrer Downstream-Cluster zu erleichtern und Daten in die Observability-Umgebung zu integrieren.

Zunächst sollten Sie zum folgenden Bildschirm geleitet werden. Um hierher zu gelangen, öffnen Sie im SUSE Cloud Observability UI das Hauptmenü, indem Sie oben links auf dem Bildschirm klicken und zu `StackPacks` > `Kubernetes` gehen.

image::integrating_first_cluster_stackpacks.png[SUSE Cloud Observability - StackPacks]

Sobald Sie sich im Kubernetes StackPack-Bildschirm befinden, ist es sehr einfach, einen Cluster zu Ihrer SUSE Cloud Observability-Umgebung hinzuzufügen.

Geben Sie einen Namen für den Cluster ein, den Sie beobachten möchten. Er muss nicht mit dem im 'kubeconfig' verwendeten Clusternamen übereinstimmen.

Der Kubernetes-Clustername muss mit einem Kleinbuchstaben oder einer Ziffer beginnen und enden und darf nur aus Kleinbuchstaben, Ziffern, Punkten und Bindestrichen (. -) bestehen.

Geben Sie den Namen ein und klicken Sie auf die `Install` Schaltfläche.  Dies sollte Sie zum folgenden Bildschirm führen, auf dem der Cluster als 'Warten auf Daten' markiert ist.

image::integrating_first_cluster_eks.png[SUSE Cloud Observability - Hinzufügen eines Clusters]

Klicken Sie auf den Clusternamen, um diesen Abschnitt zu erweitern und eine Reihe von Informationen anzuzeigen, einschließlich der Voraussetzungen und Befehle, die Sie für Ihre vorhandenen Cluster verwenden können, um sie in Ihre SUSE Cloud Observability-Umgebung hinzuzufügen.

Überprüfen Sie die Informationen und stellen Sie sicher, dass Sie die richtigen Berechtigungen für Ihre Kubernetes-Umgebung haben. Stellen Sie sicher, dass Sie eine unterstützte Version von Kubernetes verwenden.

Diese Befehle sind einzigartig für Ihre SUSE Cloud Observability-Implementierung und enthalten die erforderlichen API-Schlüssel und URLs.  Wählen Sie die entsprechenden Befehle für Ihren Cluster aus. Es gibt Abschnitte für EKS, RKE, generische Kubernetes-Cluster und viele mehr.

Diese Befehle installieren die SUSE Cloud Observability-Agenten und verbinden den Cluster mit Ihrer SUSE Cloud Observability-Umgebung.

Sie können die Befehle direkt aus der Benutzeroberfläche verwenden oder Sie können sie an Ihre Downstream-Cluster bereitstellen, indem Sie die im Abschnitt 'Bereitstellung des StackState-Agenten und Cluster-Agenten' in der xref:/k8s-quick-start-guide.adoc[Kurzanleitung] angegebenen Befehle befolgen.

image::integrating_first_cluster_eks_after_agent_install.png[SUSE Cloud Observability - Nach der Installation]

Nachdem der Cluster verbunden wurde, sollte ein grünes Häkchen in der SUSE Cloud Observability-Benutzeroberfläche angezeigt werden.

An diesem Punkt können Sie beginnen, Ihre Daten zu erkunden.
endif::feature-agent-install-cluster[]

=== Ändern Sie Ihre persönlichen Kontodaten

Schritt 4 aus Ihrer E-Mail besteht darin, Ihre persönlichen Informationen zu aktualisieren.  Klicken Sie auf den einzigartigen Link aus der E-Mail und fügen Sie bei Bedarf Ihre grundlegenden persönlichen Daten hinzu.  Sie können auch die 2FA-Authentifizierung aus diesem Abschnitt einrichten, falls erforderlich.

=== Erkunden Sie Ihre Daten

Um mit der Erkundung Ihrer Daten zu beginnen, öffnen Sie das Hauptmenü, indem Sie oben links auf dem Bildschirm klicken, und gehen Sie zu `Kubernetes`, um eine Liste der beobachtbaren Elemente anzuzeigen.

image::accessing_views_1.png[SUSE Cloud Observability - Ihre Daten erkunden]

Wählen Sie 'Cluster' aus dem Infrastrukturabschnitt aus, der eine Liste der überwachten Cluster anzeigen sollte. Wählen Sie Ihren Cluster aus, um eine der vielen integrierten Ansichten anzuzeigen.

image::accessing_views_2.png[SUSE Cloud Observability - Clusteransicht]

An diesem Punkt können Sie beginnen, die Daten für Ihren Cluster zu erkunden oder weitere Cluster hinzuzufügen, um Daten zu sammeln.

Für weitere Informationen zur Nutzung von SUSE Cloud Observability, einschließlich der Erstellung benutzerdefinierter Ansichten, siehe bitte die Standarddokumentation.

=== Einschränkungen von SUSE Cloud Observability

Bitte beachten Sie, dass SUSE Cloud Observability keine sofort einsatzbereiten RED-Signale (Rate, Errors und Duration) bereitstellt, sondern nur das Rate-Signal angezeigt wird.  Dieses Feature ist mit SUSE Rancher Prime verfügbar.
Kunden, die diese Signale für eine vollständige Observability-Lösung benötigen, sollten sich an SUSE wenden, um SUSE Rancher Prime zu besprechen; alternativ können diese Daten mit den OpenTelemetry-Collectoren gesammelt werden.

== Weitere Informationen

Für weitere Informationen, einschließlich der Voraussetzungen und unterstützten Kubernetes-Versionen für Ihre Plattform, siehe bitte die xref:/k8s-quick-start-guide.adoc[Kurzanleitung].
